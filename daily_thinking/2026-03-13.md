# Spatial AGI 每日思考 - 2026年3月13日

## 论文概览

今天深入分析了5篇Spatial AGI领域的前沿论文，涵盖了从感知到决策的完整技术栈：

1. **Dense Dynamic Scene Reconstruction** - 多视角动态场景重建与相机姿态估计
2. **OnFly** - 机载零样本空中视觉语言导航系统
3. **Pano360** - 全景视觉几何一致性（文档未提供）
4. **UAV-Based 3D Spectrum Sensing** - 无人机3D频谱感知与环境地图重建
5. **Lightweight LiDAR-Based UAV Tracking** - 轻量级激光雷达无人机跟踪

这些论文从不同维度探讨了**空中智能系统**在复杂3D环境中的感知、理解和决策能力，揭示了Spatial AGI从单模态向多模态、从离线向在线、从重传感器向轻量化演进的技术趋势。

## 一、今日核心洞察

### 1.1 "时空"维度的统一建模成为关键突破口

**观察**：今天的5篇论文中，有3篇（Dense Reconstruction、OnFly、Spectrum Sensing）的核心创新都围绕**时空建模**展开。

**深度分析**：

#### Dense Reconstruction的时空连接图
- **问题**：多相机动态场景重建面临尺度模糊、有限重叠、动态内容三大挑战
- **解决方案**：构建统一时空连接图 `Ω = Ω^temp ∪ Ω^spat ∪ Ω^st`
  - Ω^temp：相机内时间连续性（传统SLAM）
  - Ω^spat：相机间空间重叠（多视图几何）
  - Ω^st：跨相机跨时间的历史信息
- **本质**：将时间维度和空间维度统一建模，通过图结构捕获复杂的时空依赖关系

#### OnFly的混合记忆机制
- **问题**：长程导航中，滑动窗口记忆丢失全局上下文，KV-cache前缀不稳定
- **解决方案**：`Mₜ = [o₁, Qₜ, oₜ]` - 初始帧+关键帧+最新帧
- **关键帧维护**：去重、分段选择、前缀稳定序列化
- **本质**：在有限记忆容量下，保留最关键的时空信息，平衡全局和局部

#### Spectrum Sensing的空间相关性建模
- **问题**：3D频谱环境地图重建需要理解阴影衰落的空间分布
- **解决方案**：双指数空间相关性模型 `R_w(l_i, l_j) = e^(-q d_v) [a e^(-p₁ d_h) + (1-a) e^(-p₂ d_h)]`
- **本质**：分别建模水平和垂直方向的相关性，捕获3D空间的各向异性

**统一模式**：
```
时空统一建模 = 时间连续性 + 空间几何 + 历史信息融合
```

**启示**：
- Spatial AGI系统必须同时理解"何时"、"何地"、"如何变化"
- 时空联合建模能够提供比单维度分析更丰富的约束
- 关键在于设计高效的时空表示和数据结构（图、记忆、协方差）

### 1.2 "自适应"成为应对不确定性的核心策略

**观察**：4篇论文（OnFly、Spectrum Sensing、LiDAR Tracking、Dense Reconstruction）都采用了某种形式的**自适应机制**。

**技术演进脉络**：

#### 第一代：固定参数方法
- 传统卡尔曼滤波、固定噪声协方差
- **问题**：无法适应动态环境、数据质量变化、传感器特性

#### 第二代：场景自适应（Spectrum Sensing）
- **方法**：根据高度、带宽、仰角调整重建策略
- **机制**：
  - 高度三相趋势：识别最优飞行高度
  - 带宽效应：利用频率多样性提升精度
  - 天线校准：现场校准补偿机身效应
- **特点**：基于物理模型和统计规律的自适应

#### 第三代：在线自适应（OnFly、LiDAR Tracking）
- **OnFly的双智能体架构**：
  - 决策智能体：高频实时响应
  - 监控智能体：低频深度分析
  - **本质**：根据任务需求动态分配计算资源
- **LiDAR Tracking的自适应EKF**：
  - 过程噪声自适应：`Q_k = α·Q_{k-1} + (1-α)·K_k·d_k·d_k^T·K_k^T`
  - 测量噪声自适应：`R_k = β·R_{k-1} + (1-β)·(ε_k·ε_k^T + H·P_k^-·H^T)`
  - **本质**：基于创新和残差统计实时调整噪声参数

#### 第四代：混合自适应（Dense Reconstruction）
- **两阶段框架**：
  - 阶段1：鲁棒跟踪（时空连接图）
  - 阶段2：密集细化（多视图一致性）
- **宽基线初始化**：使用VGGT前馈模型提供全局一致的初始化
- **本质**：前馈模型提供鲁棒性，优化方法提供精度

**自适应的三个层次**：

| 层次 | 触发机制 | 调整对象 | 典型方法 |
|------|---------|---------|---------|
| **参数级** | 数据统计特性 | 噪声协方差、阈值 | 自适应卡尔曼滤波 |
| **模型级** | 环境特征 | 运动模型、相关性模型 | 高度自适应、场景分类 |
| **架构级** | 任务需求 | 计算资源分配、推理频率 | 双智能体、滑动窗口 |

**启示**：
- 真实世界的不确定性要求系统具备自适应能力
- 自适应应从参数级向架构级演进
- 关键在于设计有效的"触发-调整"机制和性能监控指标

### 1.3 "轻量化"与"高性能"的矛盾正在被打破

**观察**：3篇论文（OnFly、LiDAR Tracking、Dense Reconstruction）都在资源受限平台上实现了SOTA性能。

**性能对比**：

#### OnFly（Jetson Orin NX 16GB）
- 任务：视觉语言导航
- 性能：SR 67.8%（vs SPF 26.4%）
- 延迟：决策0.81s，监控1.15s
- **关键**：共享感知、混合记忆、TensorRT优化

#### LiDAR Tracking（Jetson Nano 4GB）
- 任务：无人机跟踪
- 性能：3D RMSE 2.8m（vs 粒子滤波5.5m）
- 资源：CPU 106.5%（四核），内存194.3MB
- **关键**：自适应滤波、优化聚类、恢复机制

#### Dense Reconstruction（RTX 4090）
- 任务：多相机动态场景重建
- 性能：ATE 0.011m（vs FastVGGT 0.021m）
- 内存：20.04 GB（vs FastVGGT 22.08 GB）
- **关键**：时空连接图、滑动窗口、解耦优化

**轻量化的三大策略**：

##### 策略1：计算复用（OnFly）
- **共享ViT主干**：两个智能体共享视觉编码器
- **特征缓存**：避免重复编码
- **KV-cache重用**：减少冗余注意力计算
- **效果**：监控延迟从7.47s降至1.15s（6.5倍加速）

##### 策略2：增量处理（Dense Reconstruction）
- **滑动窗口**：仅维护活动窗口的帧
- **增量式图构建**：逐帧处理而非批处理
- **解耦优化**：避免全局联合优化
- **效果**：内存使用最低（20.04 GB），支持长序列

##### 策略3：算法优化（LiDAR Tracking）
- **稀疏聚类**：DBSCAN优化参数（ε=0.5m, M_min=3）
- **自适应噪声**：减少手动调参，自动找到最优参数
- **恢复机制**：避免完全丢失后的重初始化开销
- **效果**：9.3 Hz更新频率，适合实时控制

**性能与效率的帕累托前沿正在被推进**：

```
传统观点：
  高性能 ↔ 高计算成本
  轻量化 ↔ 性能下降

新范式：
  智能算法设计 + 硬件优化 → 高性能 + 轻量化
```

**启示**：
- 轻量化不意味着牺牲性能，而是更智能的资源利用
- 关键在于识别计算瓶颈，设计针对性的优化策略
- 软硬件协同优化是未来趋势（TensorRT、CUDA Graphs、专用加速器）

### 1.4 "语义"与"几何"的融合从可选变为必需

**观察**：2篇论文（OnFly、Dense Reconstruction）的核心创新在于**语义-几何融合**。

#### OnFly的语义-几何验证器

**问题**：VLM预测的目标点仅传达语义意图，不保证几何可行性
- 可能落在障碍物上
- 可能在不可通行区域
- 纯几何修正可能违反指令语义

**解决方案**：两步验证
1. **语义引导目标修正**：
   - 计算相似度掩码：只在VLM特征相似区域内搜索
   - 连通区域分析：保留包含原始目标的区域
   - 深度可行性：避开障碍物（膨胀0.2m）
   
2. **航向感知几何门控**：
   - 目标靠近边界 → 高斯衰减前向距离
   - 引导无人机先转向，再前进

**本质**：
```
语义约束 + 几何约束 = 可执行且符合意图的目标
```

#### Dense Reconstruction的动态物体处理

**问题**：移动物体违反静态世界假设

**解决方案**：
1. **加权光流**：MegaSAM预测权重，动态物体低权重
2. **置信度学习**：细化阶段学习每流置信度图
3. **多视图约束**：同一时间戳的多视图一致性

**本质**：
```
语义分割（动态/静态） + 几何约束（多视图一致性） = 鲁棒重建
```

#### Spectrum Sensing的物理-数据混合方法

**方法**：
```
r̂^(dB) = r̂_trpl^(dB) + ŵ
```
- r̂_trpl：物理模型（两射线路径损耗）
- ŵ：数据驱动（Kriging/GPR预测阴影衰落）

**本质**：
```
物理几何（传播模型） + 数据驱动（统计建模） = 高精度REM
```

**融合的三种模式**：

| 模式 | 语义来源 | 几何来源 | 融合方式 | 典型应用 |
|------|---------|---------|---------|---------|
| **串行融合** | VLM预测 | 深度验证 | 语义→几何修正 | OnFly |
| **并行融合** | 语义分割 | 多视图几何 | 加权融合 | Dense Reconstruction |
| **层次融合** | 物理模型 | 统计学习 | 残差建模 | Spectrum Sensing |

**启示**：
- 纯几何方法缺乏高层理解，纯语义方法缺乏可执行性
- 语义-几何融合是实现Spatial AGI的关键
- 融合方式应根据任务需求选择：串行、并行或层次

### 1.5 "恢复机制"从边缘功能变为核心设计

**观察**：3篇论文（OnFly、LiDAR Tracking、Dense Reconstruction）都将**恢复机制**作为核心设计。

#### OnFly的三状态监控

**状态定义**：
- CONTINUE：正常执行
- STOP：任务完成
- LOST：偏离轨迹 → **恢复程序**

**恢复策略**：
1. 无人机先停止
2. 重新定向到上一个正常点的航向
3. 尝试恢复跟踪

**关键**：LOST状态不只是失败信号，而是触发恢复的主动机制

#### LiDAR Tracking的遮挡处理

**问题**：LiDAR回波稀疏导致间歇检测

**恢复机制**：
1. **状态传播**：无有效检测时，`x̂_k^+ = x̂_k^-`，`P_k^+ = P_k^-`
2. **动态调整**：增加过程噪声协方差反映增加的不确定性
3. **重新初始化**：t_occ时间内无测量 → 重新初始化

**关键**：在遮挡期间保持跟踪连续性，避免完全丢失

#### Dense Reconstruction的宽基线初始化

**问题**：多相机配置中，不同视点图像重叠有限

**恢复策略**：
1. **VGGT全局初始化**：提供粗略但全局一致的初始姿态
2. **单目深度对齐**：通过仿射变换对齐到VGGT尺度
3. **Bundle Adjustment细化**：进一步优化

**关键**：即使初始姿态有噪声，也能恢复

**恢复机制的设计原则**：

1. **早期检测**：
   - OnFly：LOST状态检测
   - LiDAR Tracking：遮挡检测
   - Dense Reconstruction：初始化失败检测

2. **降级策略**：
   - 从高级功能降级到基础功能
   - 从精确估计降级到粗略估计
   - 从多传感器降级到单传感器

3. **自动恢复**：
   - 无需人工干预
   - 基于历史信息重建状态
   - 平滑过渡避免跳变

**启示**：
- 真实世界系统必须假设失败会发生
- 恢复机制应从"边缘情况"变为"核心设计"
- 好的恢复机制能够显著提高系统鲁棒性和用户体验

## 二、技术演进分析

### 2.1 感知层：从单一传感器到多模态融合

**演进路径**：

```
单传感器（相机/LiDAR）
  ↓
多传感器松耦合（相机+LiDAR，独立处理）
  ↓
多传感器紧耦合（联合优化，如视觉惯性里程计）
  ↓
多模态语义融合（几何+语义+物理）
```

**典型案例**：

| 论文 | 传感器配置 | 融合方式 | 创新点 |
|------|-----------|---------|--------|
| **Dense Reconstruction** | 多相机RGB | 几何融合（时空连接图） | 时间+空间+跨相机约束 |
| **OnFly** | RGB-D + LiDAR | 语义-几何验证 | VLM意图+深度可行性 |
| **Spectrum Sensing** | 频谱传感器 | 物理-数据混合 | 传播模型+统计学习 |
| **LiDAR Tracking** | LiDAR | 自适应滤波 | 噪声协方差在线调整 |

**趋势预测**：
- **短期（1-2年）**：视觉-LiDAR-IMU三模态融合成为标配
- **中期（3-5年）**：引入频谱、热红外、事件相机等新型传感器
- **长期（5年以上）**：全频谱、全天候、全环境的多模态感知

### 2.2 建图层：从静态地图到动态语义地图

**演进路径**：

```
静态几何地图（点云、网格）
  ↓
静态语义地图（物体级标注）
  ↓
动态几何地图（4D重建，3D+时间）
  ↓
动态语义地图（运动物体+语义标签+行为预测）
```

**今天的论文覆盖**：

1. **Dense Reconstruction**：动态几何地图
   - 重建动态场景的3D结构
   - 识别并处理动态物体
   - 但缺乏语义标签

2. **Spectrum Sensing**：3D无线环境地图（REM）
   - 建模频谱占用的空间分布
   - 考虑高度维度的变化
   - 是一种特殊的"物理属性地图"

3. **OnFly**：隐式语义地图
   - 通过VLM理解环境语义
   - 但不显式构建地图，而是实时推理
   - 代表了"无地图导航"范式

**缺失的技术**：
- 今天的论文没有涉及**显式的动态语义地图构建**
- 这是Spatial AGI的重要方向：不仅要重建几何，还要理解"是什么"、"在做什么"、"将要做什么"

**未来方向**：
- **4D语义场景重建**：3D几何 + 时间 + 语义 + 运动
- **交互式地图更新**：增量式、在线更新
- **预测性地图**：基于历史预测未来状态

### 2.3 决策层：从规则驱动到学习驱动再到混合驱动

**演进路径**：

```
规则驱动（传统路径规划、控制）
  ↓
学习驱动（深度强化学习、模仿学习）
  ↓
混合驱动（学习提供策略，规则提供安全约束）
  ↓
语言驱动（自然语言指令直接驱动行为）
```

**今天的论文展示**：

#### LiDAR Tracking：规则驱动
- 基于卡尔曼滤波的跟踪算法
- 固定的状态空间模型
- 自适应机制仍基于统计规则

#### OnFly：语言驱动
- **核心创新**：零样本视觉语言导航
- **机制**：
  1. 任务管理器分解指令：L → {L₀, L₁, ..., Lₙ}
  2. VLM预测目标：`(px, py) = VLM(P_dec, Lᵢ, Oₜ)`
  3. 规划器生成轨迹：几何约束 + 动力学约束
- **本质**：语言 → 语义目标 → 几何轨迹 → 控制指令

#### Dense Reconstruction：混合驱动
- 前馈模型（VGGT）：学习驱动，提供鲁棒初始化
- Bundle Adjustment：规则驱动，提供精确优化

**语言驱动的意义**：
- **零样本泛化**：无需任务特定训练
- **人机协作**：自然交互界面
- **快速部署**：无需重新编程

**挑战**：
- VLM的可靠性（幻觉、偏见）
- 指令的模糊性和歧义性
- 语言到行动的精确映射

### 2.4 系统层：从离线处理到在线实时

**演进路径**：

```
离线批处理（数据采集 → 云端处理 → 结果返回）
  ↓
近实时处理（边缘计算，秒级延迟）
  ↓
在线实时处理（机载计算，毫秒级延迟）
  ↓
预测性处理（基于预测提前计算，零延迟感知）
```

**今天的论文覆盖**：

| 论文 | 计算位置 | 延迟 | 实时性 | 关键技术 |
|------|---------|------|--------|---------|
| **OnFly** | 机载（Jetson Orin NX） | 决策0.81s，监控1.15s | 在线实时 | TensorRT、AWQ量化、特征缓存 |
| **LiDAR Tracking** | 机载（Jetson Nano） | 9.3 Hz更新 | 在线实时 | 自适应滤波、优化聚类 |
| **Dense Reconstruction** | 边缘（RTX 4090） | 未明确（推测分钟级） | 近实时 | 滑动窗口、增量处理 |
| **Spectrum Sensing** | 未明确（推测离线） | - | 离线分析 | Kriging、GPR、矩阵补全 |

**实时化的三大挑战**：

1. **计算资源限制**：
   - 无人机载荷有限（通常<2kg）
   - 功耗限制（通常<50W）
   - 散热限制（无主动散热）

2. **算法复杂度**：
   - VLM推理：数十亿次浮点运算
   - 3D重建：O(N²)甚至O(N³)复杂度
   - 优化问题：迭代收敛慢

3. **延迟约束**：
   - 飞行控制：通常要求>50Hz
   - 避障决策：<100ms
   - 任务规划：<1s

**解决方案**（从今天的论文中总结）：

- **算法层面**：
  - 增量式处理（Dense Reconstruction）
  - 自适应调整计算频率（OnFly双智能体）
  - 稀疏化处理（LiDAR Tracking的DBSCAN）

- **系统层面**：
  - 模型压缩（OnFly的AWQ量化）
  - 特征缓存（OnFly的ViT特征缓存）
  - 硬件加速（TensorRT、CUDA Graphs）

- **架构层面**：
  - 边缘-云端协同（云端训练，边缘推理）
  - 模型蒸馏（大模型→小模型）
  - 专用加速器（NPU、TPU）

**未来趋势**：
- **预测性计算**：基于运动预测提前计算下一时刻的感知结果
- **事件驱动**：仅在环境变化时触发计算
- **分层计算**：高频低精度 + 低频高精度

## 三、跨论文关联与技术互补性

### 3.1 Dense Reconstruction ↔ OnFly：感知与决策的协同

**潜在协同**：

#### 场景1：动态环境中的语言导航

**问题**：OnFly在动态环境中（如人群密集区）可能失效
- VLM难以区分静态背景和动态障碍物
- 缺乏对动态物体运动的预测

**解决方案**：集成Dense Reconstruction的动态场景重建
1. **前端**：Dense Reconstruction实时重建动态场景
   - 识别静态背景和动态物体
   - 估计动态物体的运动轨迹
   
2. **中端**：OnFly的VLM理解语义指令
   - 在静态背景上规划路径
   - 避开动态物体的预测轨迹
   
3. **后端**：滚动时域规划器生成轨迹
   - 几何约束：静态障碍物
   - 动态约束：预测的动态物体位置

**技术接口**：
```
Dense Reconstruction输出：
  - 静态场景点云 P_static
  - 动态物体列表 {O_i, v_i, trajectory_i}

OnFly输入：
  - P_static → 规划可行空间
  - {O_i, v_i, trajectory_i} → 动态避障约束
```

#### 场景2：多无人机协作建图与导航

**问题**：Dense Reconstruction需要多相机输入，但单无人机视角有限

**解决方案**：多无人机协同
1. **建图阶段**：多无人机从不同视角采集数据
   - 每个无人机运行Dense Reconstruction的轻量版本
   - 融合多无人机的重建结果
   
2. **导航阶段**：基于重建地图进行语言导航
   - OnFly利用融合的3D地图进行路径规划
   - 实时更新地图（增量式）

**挑战**：
- 通信带宽限制（点云数据量大）
- 时钟同步（多无人机数据融合）
- 计算负载分配（边缘计算 vs 云端融合）

### 3.2 LiDAR Tracking ↔ OnFly：多传感器融合的跟踪与导航

**潜在协同**：

#### 场景1：目标跟踪与跟随

**问题**：OnFly的语言导航可能需要跟踪特定目标（如"跟随那个人"）

**解决方案**：集成LiDAR Tracking
1. **语言理解**：OnFly的VLM解析指令，识别目标
2. **视觉跟踪**：初始阶段使用视觉跟踪
3. **LiDAR跟踪**：当视觉失效（低光照、遮挡）时，切换到LiDAR Tracking
4. **融合跟踪**：视觉+LiDAR联合估计目标位置

**技术接口**：
```
OnFly输出：
  - 目标语义描述："穿红色衣服的人"

LiDAR Tracking输入：
  - 目标初始位置
  - LiDAR点云流

LiDAR Tracking输出：
  - 目标3D位置（实时更新）
  - 速度估计
  - 跟踪质量指标

OnFly输入：
  - 目标3D位置 → 规划跟随轨迹
  - 跟踪质量 → 决定是否切换传感器
```

#### 场景2：集群协调与相对定位

### 3.3 Spectrum Sensing ↔ OnFly：频谱感知辅助的通信感知导航

**潜在协同**：

2. **REM构建**：动态更新3D无线环境地图
3. **路径规划**：OnFly规划路径时，考虑通信质量约束

**技术接口**：
```
Spectrum Sensing输出：

  - 信号强度预测
  - 干扰源位置

OnFly输入：
  - 通信约束：信号强度 > 阈值
  - 修改规划目标：
    原目标：min 路径长度
    新目标：min 路径长度 + λ·Σ(1/信号强度)
```

#### 场景2：动态频谱管理

**问题**：多无人机集群通信时，频谱资源有限

**解决方案**：
1. **频谱感知**：实时监控频谱占用
2. **动态分配**：根据REM动态分配频谱资源
3. **干扰避免**：OnFly规划路径时，避开高干扰区域

**应用**：
- 大型活动（演唱会、体育赛事）的多无人机直播
- 灾难救援的多无人机协同搜索

## 五、个人见解与批判性思考

### 5.1 技术层面的批判性思考

#### 思考1：VLM的可靠性问题被低估

**观察**：OnFly依赖VLM进行零样本导航，性能显著优于传统方法（SR 67.8% vs 26.4%）

**批判**：
1. **成功率的绝对值仍然不高**：
   - 67.8%的成功率意味着约1/3的任务失败
   - 在安全关键应用（如救援、物流）中不可接受
   - 论文未详细分析失败案例

2. **VLM的已知问题未充分讨论**：
   - **幻觉**：VLM可能"看到"不存在的东西
   - **偏见**：训练数据的偏见影响泛化
   - **对抗攻击**：小扰动可能导致错误决策
   - **分布外泛化**：在训练数据外的环境性能未知

3. **缺乏不确定性量化**：
   - VLM输出的置信度如何？
   - 如何检测VLM的不可靠输出？
   - 如何设计fallback机制？

**建议**：
- 引入**校准的置信度估计**：不仅输出结果，还输出置信度
- 设计**多模型投票机制**：多个VLM投票，检测异常
- 建立**人机协同**：在低置信度时请求人类确认
- 开发**VLM测试集**：专门测试VLM在导航任务中的可靠性

#### 思考2：实时性与精度的权衡缺乏理论指导

**观察**：今天的论文都实现了某种形式的轻量化，但缺乏系统的权衡分析

**批判**：
1. **参数选择的经验性**：
   - OnFly：监控频率2s/次，关键帧预算S=4
   - LiDAR Tracking：DBSCAN ε=0.5m, M_min=3
   - 这些参数如何选择？缺乏理论指导

2. **权衡的帕累托前沿未探索**：
   - 延迟 vs 精度的权衡曲线
   - 计算资源 vs 性能的权衡曲线
   - 不同应用场景的最优工作点

3. **缺乏自适应参数调整**：
   - 参数大多是固定的
   - 为何不根据任务难度、环境复杂度自适应调整？

**建议**：
- 建立**理论模型**：量化延迟-精度权衡
- 开发**自动调参算法**：基于贝叶斯优化、强化学习
- 提供**配置指南**：针对不同应用场景的推荐配置

#### 思考3：恢复机制的有效性验证不足

**观察**：3篇论文都提出了恢复机制，但验证不充分

**批判**：
1. **压力测试不足**：

   - OnFly：未专门测试LOST状态的恢复成功率
   - Dense Reconstruction：未测试初始化失败的恢复

### 5.2 方法论层面的批判性思考

#### 思考4：数据集的多样性和规模不足

**观察**：今天论文的实验数据集规模相对较小

| 论文 | 数据集规模 | 环境 | 来源 |
|------|-----------|------|------|

| LiDAR Tracking | 未明确数量（推测数十个轨迹） | 户外 | 自采 |

**批判**：
1. **场景多样性不足**：
   - 缺乏城市、室内、森林、海洋等多样化环境
   - 缺乏极端天气（雨、雪、雾）测试
   - 缺乏夜间、低光照测试

2. **规模不足以训练深度学习**：
   - 深度学习需要海量数据
   - 当前数据集规模难以支撑大规模预训练

3. **缺乏公开基准**：
   - 多数数据集为自采，未公开
   - 难以进行公平比较
   - 阻碍社区发展

**建议**：
- 建立**大规模公开数据集**：
  - 涵盖多样化环境
  - 包含多传感器数据
  - 提供标注和基准
  
- 推进**数据集标准化**：
  - 统一数据格式
  - 统一评估指标
  - 建立排行榜

#### 思考5：评估指标不够全面

**观察**：评估主要关注精度和效率，缺乏其他重要维度

**批判**：
1. **缺乏鲁棒性指标**：
   - 在对抗攻击下的性能
   - 在传感器故障下的性能
   - 在极端环境下的性能

2. **缺乏安全性指标**：
   - 碰撞率（OnFly有，但其他论文没有）
   - 失败模式分析
   - 安全边界

3. **缺乏用户体验指标**：
   - 恢复时间（用户等待时间）
   - 可解释性（用户理解程度）
   - 交互自然度

4. **缺乏长期性能指标**：
   - 系统退化
   - 维护需求
   - 能耗

**建议**：
- 建立**多维评估框架**：
  ```
  性能 = 精度 + 效率 + 鲁棒性 + 安全性 + 用户体验
  ```
  
- 开发**场景化评估**：
  - 不同应用场景的评估重点不同
  - 提供场景化的评估套件

  - 潜在风险

  - 在线匿名化（模糊人脸、车牌）

   - 无人机制造、运行、维护的能源消耗
   - 碳排放

**建议**：
- 研究**环境友好型设计**：
  - 低噪声设计
  - 能源高效算法
  - 可回收材料

4. **语义-几何融合**：结合高层理解和物理约束
5. **恢复机制**：从边缘功能变为核心设计

- 单传感器 → 多模态融合 → 语义感知

- 静态地图 → 动态地图 → 语义地图

**决策层**：
- 规则驱动 → 学习驱动 → 语言驱动

**系统层**：
- 离线处理 → 在线实时 → 预测性处理

1. 轻量化多模态融合

**长期挑战（5年以上）**：

**对于研究者**：
1. **关注系统工程**：不仅是算法创新，还要考虑部署

今天的5篇论文展示了Spatial AGI领域的蓬勃发展和巨大潜力。从多相机动态重建到零样本语言导航，从3D频谱感知到轻量级跟踪，这些工作正在构建未来自主智能系统的技术基石。
