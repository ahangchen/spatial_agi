# Scale Can't Overcome Pragmatics: The Impact of Reporting Bias on VLM Reasoning

**发表日期**: 2026-02-26
**arXiv**: https://arxiv.org/abs/2602.23351v1
**作者**: Amita Kamath, Jack Hessel, et al.
**分析方法**: GLM WebReader
**日期**: 2026-03-02

---

## 核心问题

VLM在标准化基准上表现优异，但在需要计数、空间推理和组合推理的任务上经常失败。为什么规模化（数据、模型、多语言）无法解决这些问题？

---

## 主要发现

### 报告偏差（Reporting Bias）

**定义**: 人们在描述图像时系统性忽略某些类型的信息

**四种被忽略的推理技能**:
1. **空间推理**: "在左边"、"在上方"
2. **时间推理**: "之前"、"之后"
3. **计数**: "37个人"、"3个物体"
4. **否定**: "不是"、"没有"

### 数据分析结果

**LAION数据集**:
- 空间推理出现率: **0.1%**
- 计数出现率: **0.3%**
- 时间推理: **极少**
- 否定: **极少**

**原因**: 人们写描述时会省略"显而易见"的信息

## Q&A记录

### Q1: 核心发现是什么？

**规模化无效**:
- ❌ 数据规模化 → 不改善推理
- ❌ 模型规模化 → 不改善推理
- ❌ 多语言扩展 → 不改善推理

**根本原因**: 人类行为导致的报告偏差不随规模改变

**解决方案**: 专门的数据收集（精心设计的标注指令）

### Q2: 与Spatial AGI的关系？

**直接启发**:
1. **数据策略 > 模型架构**: Spatial AGI需要专门的空间推理数据
2. **规模化不是万能**: 不能指望自动涌现空间推理能力
3. **显式标注重要**: 需要明确标注空间关系

**对Spatial AGI的三个关键教训**:

**教训1: 数据质量 > 数据数量**
```
传统思路: 更多数据 → 更好性能
现实: 专门数据 > 泛化数据
```

**教训2: 需要主动收集空间推理数据**
```
策略:
- 专门的空间关系标注
- 几何约束的显式描述
- 空间推理问题的QA对
```

**教训3: 不能依赖规模化自动涌现**
```
Spatial AGI必须:
- 设计专门的空间推理数据集
- 主动收集而非被动等待
- 显式训练空间推理能力
```

### Q3: 如何改进标注指令？（自由问题）

**发现**: LLaVA和PixMo的指令比COCO多2-3倍的空间推理和计数

**关键改进**:
1. **明确要求**: "描述物体之间的空间关系"
2. **具体提示**: "使用'左边'、'右边'、'上方'等词"
3. **数量要求**: "准确计数物体数量"

**对Spatial AGI的应用**:
```
标注指令应该包含:
- 空间位置描述（坐标、相对位置）
- 空间关系描述（包含、连接、相邻）
- 空间变化描述（移动、旋转、变形）
- 几何约束描述（距离、角度、碰撞）
```

## 个人思考

### 最有趣的发现

**0.1%的空间推理出现率**令人震惊！这说明我们浪费了99.9%的数据在无关信息上。

### 最意外的结果

**多语言扩展也无效**。我原本以为不同语言可能有不同的描述习惯，但报告偏差是普遍现象。

### 最有价值的启发

**数据策略决定上限**。无论模型多大、数据多海量，如果数据本身缺乏空间推理信息，模型就无法学会空间推理。

### 对Spatial AGI架构的影响

**新增原则1**: 数据策略 > 模型架构
**新增原则5**: 评估即学习

这篇论文强化了"数据质量 > 数据数量"的观点，并证明Spatial AGI必须主动收集专门的空间推理数据。

## 实验结果

- **VLM性能**: 平均落后人类54分（满分100）
- **规模化**: CLIP需要不可行的数据量才能达到人类水平
- **多语言**: 无改善
- **专门数据**: 有效（提升2-3倍）
