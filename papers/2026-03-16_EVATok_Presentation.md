## 1️⃣ 背景与动机

### 问题：传统视频Tokenizer的低效

**固定长度分配的局限**：
- 所有视频使用相同数量的tokens
- 简单/静态场景：浪费计算资源
- 复杂/动态场景：tokens不足，质量下降

**现实世界的多样性**：
```
简单场景（走廊）:
- 少量变化
- 重复模式
- 需要: 64-128 tokens

复杂场景（繁忙街道）:
- 多个运动物体
- 复杂布局
- 需要: 256-512 tokens
```

---

## 2️⃣ 核心创新

### 创新1: Proxy Reward - 最优分配的形式化

**首次明确定义"最优"**：
```
R_proxy = w_q × Quality - w_l × Length

目标: 找到最大化R的token分配
```

**优势**：
- 可优化的数学目标
- 支持质量-成本权衡
- 替代启发式方法

---

### 创新2: 四阶段训练框架

```
Stage 1: Proxy Tokenizer
→ 学习在所有分配下重建

Stage 2: 数据集策划
→ 搜索每个视频的最优分配

Stage 3: Router训练
→ 轻量级模型快速预测

Stage 4: Final Tokenizer
→ 在最优分配下专用训练
```

**关键洞察**: 消除Training-Inference Gap

---

### 创新3: 视频语义增强

**双重机制**：
1. **Representation Alignment**
   - 对齐V-JEPA2语义特征
   - 注入空间先验知识

2. **VideoMAE Discriminator**
   - 语义级别反馈
   - 减少模糊和闪烁

**效果**: rFVD提升52%，gFVD提升28%

---

## 3️⃣ 技术方案

### 架构：1D Q-Former Tokenizer

**输入处理**:
```
16×128×128 视频
↓ Patchify (8×空间, 4×时间)
4×16×16 3D Embeddings
↓ 根据分配初始化
1D Query序列 (32-512 tokens/块)
```

**编码-解码**:
- Q-Former编码器 + 向量量化
- Q-Former解码器 + 视频重建
- 时间因果注意力

---

### Router: 快速最优分配预测

**设计**:
- ViT-S架构 (19.9M参数)
- 100k视频训练
- 单次前向传播预测

**泛化能力**:
- 在WebVid训练，UCF测试
- 接近理论最优性能
- 节省42% tokens (UCF)

## 4️⃣ 实验结果

### 重建质量 (UCF-101)

| 方法 | rFVD↓ | Token节省 |
