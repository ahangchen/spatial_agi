## 核心信息

### 摘要

当前的视频生成模型无法模拟3D动作（如力和机器人操作）的物理后果，因为它们缺乏对动作如何影响3D场景的结构性理解。本文提出**RealWonder**，这是首个从单张图像进行实时动作条件视频生成的系统。

**核心洞察**：使用**物理仿真作为中间桥梁**——不是直接编码连续动作，而是通过物理仿真将它们转换为视频模型可以处理的视觉表示（光流和RGB）。

RealWonder集成了三个组件：
1. **3D重建**：从单张图像重建3D场景
2. **物理仿真**：预测动作后果
3. **蒸馏视频生成器**：仅需4个扩散步骤

**性能**：在480×832分辨率下达到**13.2 FPS**，支持刚性物体、可变形体、流体和颗粒材料的交互式探索（力、机器人动作、相机控制）。

**应用前景**：沉浸式体验、AR/VR、机器人学习

**开源**：代码和模型权重已公开发布

---

### 关键贡献

1. **首个实时流式系统**：提出RealWonder，第一个接受3D物理动作（包括力、扭矩、机器人动作、相机控制）作为输入的实时流式视频生成系统，在480×832分辨率下达到13.2 FPS

2. **创新的中间表示架构**：引入使用物理仿真作为中间表示的新架构，无需在动作-视频对上训练或对动作进行标记化，即可预测3D物理动作后果

3. **光流条件蒸馏方案**：设计了将光流条件整合到视频生成中的蒸馏方案，显著减少扩散开销同时实现有效的流控制

#### 1) 核心思想和动机

**核心挑战**：

当前视频生成模型面临一个根本性的不匹配：
- **物理动作的运作方式**：在力和运动学空间中操作（连续、高维、无界）
- **视频模型的处理方式**：在像素或潜在空间中处理视觉模式

具体问题：
1. **连续动作难以标记化**：力和扭矩等动作是连续且无界的，无法使用适用于离散输入（如相机姿态）的标记化方案
2. **训练数据稀缺**：获取动作-视频对是一个开放性问题，从视频中推断导致观察到的运动的精确物理动作通常是不可行的

**关键洞察**：

引入**物理仿真作为中间表示桥梁**：
- 物理仿真器将动作后果转换为**视觉运动模式**（光流和粗糙RGB预览）
- 这些中间表示：
  - 保留了动作与结果之间的**因果关系**
  - 提供了现代视频生成器设计用于处理的**视觉信号**
  - 可实时计算

**核心优势**：
- 优雅地绕过了标记化问题：通过物理仿真器将连续动作信号转换为离散像素
- 不需要动作-视频对，只需要**2D流-视频对**来训练视频生成器

---

#### 2) 主要技术方法

##### A. 三阶段流水线架构

RealWonder采用三阶段流水线，使用物理仿真作为中间表示桥梁：

```
输入图像 + 动作序列
    ↓
[阶段1] 3D场景重建
    → 估计几何和材料属性
    ↓
[阶段2] 物理仿真
    → 计算场景对动作的动态响应
    → 渲染为光流F_t和粗糙RGB预览Ṽ_t
    ↓
[阶段3] 蒸馏视频生成
    → 4步扩散生成真实感视频
    ↓
输出视频流 V_t (13.2 FPS)
```

##### B. 输入表示

**输入**：
- 单张RGB图像：I ∈ ℝ^(H×W×3)
- 3D物理动作序列：{a_t}, t=1,2,⋯

**三种动作类型**：
1. **外力**：f_t(x,y,z) ∈ ℝ³，在3D位置施加
2. **机器人末端执行器命令**：r_t = {p_t^ee, q_t^ee, g_t}
   - p_t^ee: 位置
   - q_t^ee: 方向
   - g_t: 夹持器状态
   - 支持预定义机器人模型（如Franka）
3. **相机姿态**：C_t = {R_t, t_t}
   - R_t: 旋转
   - t_t: 平移

**输出**：
- 视频流：V_t，在交互速率（最高13.2 FPS，480×832分辨率）生成

---

#### 3) 算法流程和关键步骤

##### 阶段1：单图像3D场景重建

**场景表示**：S = B ∪ O
- B：静态背景
- O：动态物体

**A. 背景重建**：

表示为点云：B = {(p_i^B, c_i^B)}_{i=1}^{N_B}
- p_i^B ∈ ℝ³：3D位置
- c_i^B ∈ ℝ³：RGB颜色

重建流程：
1. 分割静态区域
2. 修复被物体遮挡的区域
3. 估计每像素深度
4. 反投影到3D空间
5. 作为静态碰撞边界

**B. 物体重建**：

表示为点云：O = {(p_j^O, c_j^O, v_j)}_{j=1}^{N_O}
- p_j^O ∈ ℝ³：3D位置
- c_j^O ∈ ℝ³：RGB颜色
- v_j ∈ ℝ³：速度

重建流程：
1. 从反投影像素导出点（类似背景）
2. 使用前馈重建模型生成完整3D网格
3. 通过姿态估计和尺度对齐将网格注册到场景坐标系
4. 提取不可见表面的网格顶点（如物体背面）
5. 补充反投影的物体像素，确保捕获完整的物体几何

**C. 材料估计**：

使用视觉-语言模型（VLM）：
- 分类为6种材料类别：刚性、弹性、布料、烟雾、液体、颗粒
- 估计物理参数 m：密度、摩擦系数、弹性模量、粘度（取决于材料类型）
- 用户可覆盖分类和参数以进行创意控制或提高物理准确性

---

##### 阶段2：物理仿真作为中间桥梁

**A. 动作表示**：

1. **外力**：f_t(x,y,z) - 直接在指定3D位置施加
2. **机器人命令**：通过逆运动学（IK）转换为关节扭矩和力，然后在仿真中驱动关节机器人模型
3. **相机姿态**：在渲染期间应用

**B. 物理求解器**：

每个时间步t，物理引擎接收：
- 当前场景状态 S_t
- 动作 a_t

计算更新：
```
(p_{t+1}, v_{t+1}) = PhysicsStep(S_t, a_t)
```

**专用求解器**：
- **刚体动力学**：通过形状匹配进行碰撞处理
- **弹性体、布料、烟雾**：基于位置的动力学（PBD）
- **液体和颗粒材料**：物质点法（MPM）

**耦合处理**：多材料交互

**性能**：单个物理步骤通常在不到2ms内完成

**C. 中间表示生成**：

**1. 光流计算**：

计算像素空间流：F_t ∈ ℝ^(H×W×2)

公式：
```
F_t(u,v) = Π(p_t + Δt·v_t) - Π(p_t)
```

其中：
- Π：相机投影
- (u,v)：像素坐标
- 捕获应用动作的运动后果

**2. 粗糙RGB渲染**：

渲染预览视频：Ṽ_t ∈ ℝ^(H×W×3)

使用简单点云光栅化：
- 视觉上近似
- 提供关键的结构线索（如遮挡变化）
- 纯流无法捕获

**三个关键目标**：
1. 保留动作与视觉后果之间的因果关系
2. 存在于视频模型可以处理的视觉域中
3. 可以实时计算

---

##### 阶段3：实时条件视频生成

**A. 流条件教师模型**：

**基础模型**：预训练的图像到视频扩散模型 G_base(text, I)

**流控制增强**（通过后训练）：

1. **基于流的噪声扭曲**：
   - 采样单帧高斯噪声：z ~ N(0,I)
   - 根据流场时间扭曲：z^F = Warp(z, F)
   - 扭曲保留高斯分布，同时将运动模式直接编码到噪声结构中

2. **流匹配目标微调**：
   - 建模流扭曲噪声与数据分布之间的速度场
   - 不需要额外的嵌入模块或架构更改
   - 通过初始噪声直接注入控制

**B. 因果蒸馏用于流式传输**：

**挑战**：流条件教师仍是需要完整序列处理的双向模型，与实时流式生成不兼容

**解决方案**：蒸馏为因果学生模型，仅用4个去噪步骤顺序生成帧

**方法**：分布匹配蒸馏（DMD）

目标函数：
```
∇L_DMD = E_t[∇_θ KL(p_fake,t || p_real,t)]
```

其中 p_fake,t 表示学生在时间t的估计分布

**稳定长视野生成**：

1. **Self Forcing训练范式**：训练期间的自回归展开
2. **修复质量退化**：
   - 在应用RoPE之前存储KV缓存
   - 添加注意力汇（attention sink）
   - 类似于并发工作

**C. 流式推理**：

**RGB条件通过SDEdit**：

在4步去噪过程中合并粗糙RGB预览 Ṽ_t：

不是从纯流扭曲噪声 V_{t,(4)} = z_t^F 一直去噪到干净样本 x_0，而是从混合开始：

```
V_{t,(3)} = α_(3)·E(Ṽ_t) + √(1-α_(3)²)·z_t^F
```

其中：
- E：VAE编码器
- α_(3)：扩散步骤3的噪声调度系数

**优势**：
- 从步骤3而非步骤4开始去噪
- 允许模型一个初始步骤去噪混合
- 执行剩余三个标准去噪步骤

**双条件**：
- 保留来自流的运动准确性
- 合并来自物理预览的结构线索
- 导致连贯的物体变形和遮挡处理

**流式架构**：

维护两个并行流：
1. 物理仿真及其渲染为中间表示
2. 视频生成以13.2 FPS运行，消费最新的物理导出条件

因果模型G的帧生成：
```
V_{t+1} = G(text, I, F_{t+1}, Ṽ_{t+1}, {V_j}_{j≤t})
```

#### 4) 实现细节

**A. 物理仿真**：

- **仿真器**：Genesis
- **时间步**：0.01s
- **子步骤**：每个仿真步骤最多20个子步骤（数值稳定性）
- **流速率**：仿真和渲染流以30 FPS并行运行
- **机器人**：使用Genesis提供的Franka机器人模型，支持机器人与多种材料之间的交互

**B. 视频模型训练**：

**基础模型**：VideoXFun wan2.1-1.3B-InP（I2V基础模型）

**流条件后训练**：
- 冻结所有权重
- 在每个注意力块中注入LoRA模块，秩为2048
- 训练300K迭代，学习率10^-5

**因果蒸馏**：
- Self Forcing风格训练
- 因果适应阶段：从后训练模型采样2K ODE轨迹，训练学生模型3K迭代（MSE损失）
- 分布匹配蒸馏：600迭代
- **总训练计算**：约128 A100 GPU天

**C. 数据集**：

遵循RAFT方法构建200K对流扭曲噪声-视频对：
- **180K真实世界视频**：来自OpenVid，过滤到80-120帧
- **20K合成视频**：由Wan2.1-14B-T2V模型生成，使用VidProM的提示

#### 1) 如何理解和表示空间

RealWonder采用了**分层混合表示**来理解和表示空间：

##### A. 点云表示

**核心思想**：使用轻量级点云表示，在最小化渲染计算开销的同时提供足够的几何细节

**背景表示**：
```
B = {(p_i^B, c_i^B)}_{i=1}^{N_B}
```
- 静态3D点 + RGB颜色
- 作为碰撞边界

**物体表示**：
```
O = {(p_j^O, c_j^O, v_j)}_{j=1}^{N_O}
```
- 动态3D点 + RGB颜色 + 速度
- 补充不可见表面的网格顶点

**优势**：
- 计算效率高（适合实时系统）
- 足以支持物理仿真
- 可从单张图像重建

##### B. 材料属性表示

**6种材料类别**：
1. **刚性**（Rigid）
2. **弹性**（Elastic）
3. **布料**（Cloth）
4. **烟雾**（Smoke）
5. **液体**（Liquid）
6. **颗粒**（Granular）

**物理参数**：
- 密度
- 摩擦系数
- 弹性模量
- 粘度

**估计方法**：视觉-语言模型（VLM）分类 + 用户可调

**意义**：不同的材料需要不同的物理求解器，这对空间理解至关重要

##### C. 3D重建流程

从2D到3D的转换：
1. **深度估计**：从单张图像估计每像素深度
2. **反投影**：将像素转换为3D点
3. **网格生成**：使用前馈重建模型
4. **姿态估计**：将网格注册到场景坐标系
5. **尺度对齐**：确保正确的物理尺度

**与Spatial AGI的关系**：
- 提供了从视觉感知到3D空间表示的桥梁
- 支持多种材料类型，增强了空间理解的丰富性
- 单图像输入降低了空间重建的门槛

#### 2) 如何处理空间关系

##### A. 物理仿真作为空间关系处理器

**核心机制**：通过物理仿真自动处理复杂的空间关系

**空间关系类型**：

1. **碰撞关系**：
   - 形状匹配算法处理刚体碰撞
   - 点云之间的距离检测
   - 碰撞响应计算

2. **遮挡关系**：
   - 通过粗糙RGB渲染捕获
   - 光流无法单独表示遮挡变化
   - 视频生成器学习处理遮挡线索

3. **力传播关系**：
   - 外力如何在3D空间中传播
   - 多材料交互的耦合处理
   - 接触力和摩擦力

4. **时间演化关系**：
   ```
   (p_{t+1}, v_{t+1}) = PhysicsStep(S_t, a_t)
   ```
   - 当前状态 + 动作 → 下一状态
   - 保持物理一致性

##### B. 多材料空间交互

**专用求解器**：
- **刚体**：形状匹配碰撞处理
- **弹性体/布料/烟雾**：基于位置的动力学（PBD）
- **液体/颗粒**：物质点法（MPM）

**耦合机制**：
- 不同材料之间的相互作用
- 例如：机器人抓取布料、力推动液体、风吹颗粒

**意义**：真实世界空间包含多种材料，Spatial AGI需要理解这些交互

##### C. 动作空间映射

**3D动作到视觉表示的转换**：

1. **外力**：
   - 3D位置 + 力向量
   - → 通过物理仿真传播到整个场景
   - → 转换为光流（2D视觉表示）

2. **机器人动作**：
   - 末端执行器命令（位置、方向、夹持器状态）
   - → 逆运动学转换为关节扭矩
   - → 驱动机器人模型
   - → 与场景交互产生运动

3. **相机控制**：
   - 相机姿态（旋转、平移）
   - → 改变视角
   - → 渲染不同角度的场景

**关键洞察**：
- 物理仿真作为"空间关系解算器"
- 自动处理复杂的3D交互
- 输出视频模型可理解的2D表示

#### 3) 对Spatial AGI的启发

##### A. 物理仿真作为认知桥梁

**核心启发**：物理仿真可以桥接符号推理和感知生成

**传统问题**：
- 符号AI：擅长推理但缺乏感知
- 神经网络：擅长感知但缺乏结构化推理

**RealWonder的解决方案**：
- 物理仿真提供结构化的中间表示
- 保留了因果关系和物理定律
- 可被神经网络（视频生成器）理解和处理

**对Spatial AGI的意义**：
- 不需要端到端学习物理
- 可以利用成熟的物理引擎
- 神经网络专注于视觉真实感

##### B. 无需动作-视频对的训练

**传统困境**：
- 动作-视频对数据稀缺
- 从视频推断物理动作困难

**RealWonder的创新**：
- 只需要流-视频对
- 物理仿真提供动作到流的转换
- 流可以从现有视频提取（RAFT）

**对Spatial AGI的意义**：
- 大幅降低数据需求

- 更容易扩展到新场景

##### C. 实时交互能力

**技术突破**：

**对Spatial AGI的意义**：
1. **机器人应用**：
   - 实时预测动作后果

##### D. 多材料统一处理

**能力**：
- 刚性物体、可变形体、流体、颗粒材料

**对Spatial AGI的意义**：

#### 4) 如何实现13.2 FPS的实时性能？

##### A. 速度分解

**总体性能**：13.2 FPS @ 480×832

**各阶段耗时**（推测，论文未详细列出）：
1. 3D重建：一次性（~秒级）
2. 物理仿真：<2ms/步
3. 渲染光流+RGB：<10ms
4. 视频生成（4步扩散）：~60ms
5. 其他开销：~10ms

**总计**：~75ms/帧 ≈ 13.3 FPS

##### B. 关键加速技术

**1. 物理仿真加速**：
- 轻量级点云表示（而非密集网格）
- 高效求解器（Genesis引擎）
- 时间步0.01s，子步骤最多20
- 单步<2ms

**2. 扩散模型加速**：
- **4步扩散**（vs. 传统50步）
  - Distribution Matching Distillation (DMD)
  - 因果学生模型
  
- **流式生成**：
  - 因果架构（vs. 双向）
  - 帧顺序生成，无需完整序列
  
- **LoRA后训练**：
  - 不修改基础模型
  - 仅训练轻量适配器

**3. 双流并行**：
```
流1：物理仿真 + 渲染（30 FPS）
流2：视频生成（13.2 FPS）
```
- 流1更快，持续提供条件
- 流2消费最新条件
- 异步执行

**4. 因果生成**：
```
V_{t+1} = G(text, I, F_{t+1}, Ṽ_{t+1}, {V_j}_{j≤t})
```
- 只依赖过去帧
- 不需要未来信息
- 支持流式

**3. 物理模型简化**：

- MPM：计算密集，有截断误差
- 刚体：形状匹配是启发式

**影响**：
- 长时间仿真可能漂移
- 复杂交互可能不准确

#### 6) 其他技术细节与洞察

##### A. 训练数据策略

**200K流-视频对**：
- 180K真实（OpenVid）
- 20K合成（Wan2.1-14B-T2V）

**混合策略的好处**：
- 真实数据：真实分布
- 合成数据：多样性、可控性

**流的提取**：
- RAFT算法
- 从现有视频自动提取
- 无需人工标注

##### B. SDEdit的作用

**问题**：仅用光流，无法捕获遮挡

**解决方案**：合并粗糙RGB预览
```
V_{t,(3)} = α_(3)·E(Ṽ_t) + √(1-α_(3)²)·z_t^F
```

**效果**：
- 流：运动准确性
- RGB：结构线索（遮挡、变形）
- 互补

##### C. 因果生成的挑战

**问题**：长序列质量退化

**Self Forcing的问题**：
- 训练-测试不匹配
- 误差累积

**RealWonder的解决**：
- KV缓存（RoPE前）
- 注意力汇
- 稳定长序列生成
