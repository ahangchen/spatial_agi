## 核心信息

### 摘要

虽然大规模扩散模型已经革新了视频合成，但实现对多主体身份和多粒度运动的精确控制仍然是一个重大挑战。最近尝试弥合这一差距的方法往往受到运动粒度有限、控制模糊性和身份退化的困扰，导致身份保持和运动控制的性能欠佳。

在本研究中，我们提出了 **DreamVideo-Omni**，这是一个统一框架，通过渐进式两阶段训练范式，实现和谐的多主体定制与全运动控制。在第一阶段，我们集成全面的控制信号进行联合训练，包括主体外观、全局运动、局部动态和摄像机运动。为确保稳健和精确的可控性，我们引入了**条件感知 3D 旋转位置嵌入**来协调异构输入，以及**分层运动注入策略**来增强全局运动指导。此外，为解决多主体模糊性，我们引入了**组和角色嵌入**来明确将运动信号锚定到特定身份，有效地将复杂场景解耦为独立可控的实例。

在第二阶段，为减轻身份退化，我们通过在预训练的视频扩散骨干上训练潜在身份奖励模型，设计了**潜在身份奖励反馈学习范式**。这在潜在空间中提供运动感知的身份奖励，优先考虑与人类偏好一致的身份保持。在我们精心策划的大规模数据集和用于多主体和全运动控制评估的综合 DreamOmni Bench 的支持下，DreamVideo-Omni 在生成具有精确可控性的高质量视频方面表现出优越的性能。

### 关键贡献

1. **统一框架**: 提出 DreamVideo-Omni，首个在单一 DiT 架构中和谐集成多主体定制与全运动控制的统一框架

2. **专用架构组件**: 
   - 组和角色嵌入解决多主体模糊性
   - 条件感知 3D RoPE 协调异构输入以稳定训练
   - 分层运动注入增强全局运动控制

3. **潜在身份奖励反馈学习**: 训练基于 VDM 的潜在身份奖励模型来评估运动感知的身份保持，有效减轻大运动下的身份退化

4. **DreamOmni Bench**: 新基准，包含超过 1K 精选视频和全面注释，旨在同时量化多主体一致性和运动控制精度

5. **大规模数据集**: 建立全面的数据处理流水线，用于多主体定制和全运动控制任务

## 核心技术发现

### 1. 渐进式两阶段训练范式

DreamVideo-Omni 采用创新的两阶段训练策略，有效解决了身份保持与运动控制之间的内在权衡：

#### Stage 1: Omni-Motion and Identity Supervised Fine-Tuning (全运动和身份监督微调)

**目标**: 建立统一的可控性基础

**关键设计**:
- 集成异构控制信号：主体外观、全局运动（bounding box）、局部运动（轨迹）、摄像机运动
- 使用结构化三元组：⟨Reference Subject, Global Box, Local Trajectory⟩
- 所有信号集成到单一 DiT 架构中

**输入条件**:
1. **主体外观**: 每个主体一张参考图像，分割后获得空白背景图像
2. **全局物体运动**: 场景锚定的边界框序列
3. **局部物体运动**: 稀疏点轨迹
4. **摄像机运动**: 应用于背景像素的点轨迹

#### Stage 2: Latent Identity Reinforcement Learning (潜在身份强化学习)

**目标**: 通过与人类偏好对齐来增强身份保真度

**关键创新**:
- 训练专门的 Latent Identity Reward Model (LIRM)
- 在潜在空间中进行奖励反馈学习
- 绕过昂贵的 VAE 解码，直接在潜在空间评估

### 2. 条件感知 3D Rotary Positional Embedding (RoPE)

为在统一 DiT 架构中处理异构输入（视频潜在表示、多主体参考图像、运动控制信号），设计了专门的时间索引策略：

**时间索引分配**:
- **视频帧标记**: t ∈ [0, T-1] - 确保时间一致性
- **参考图像标记**: 共享的 t_ref - 明确解耦参考主体
- **填充标记**: 独特的 t_pad - 识别并忽略非信息性标记
- **轨迹标记**: t ∈ [0, T-1] - 与视频帧严格时空对齐

**优势**:
- 加快收敛速度
- 增强训练稳定性
- 有效协调异构输入

### 3. 分层运动注入策略

为精确的全局运动控制，采用密集注入机制：

```
h_0 = z_t + Z_in(z_box)
h_{l+1} = Block_l(h_l) + Z_l(z_box)
```

其中：
- z_t: 输入噪声视频潜在表示
- z_box: 边界框潜在表示
- Z_in 和 Z_l: 输入阶段和第 l 个 DiT 块的可学习零卷积
- h_l: 第 l 个块的输入隐藏状态

**优势**:
- 增强全局运动控制精度
- 不增加 token 序列长度
- 通过可学习零卷积实现高效注入

### 4. 组和角色嵌入

为解决多主体生成中的控制模糊性，引入两种可学习嵌入：

#### 组嵌入 (Group Embeddings)
- 将基本控制单元定义为三元组：⟨Reference Subject, Global Box, Local Trajectory⟩
- 为每个单元分配唯一的组嵌入
- 参考图像：组嵌入添加到图像的所有潜在 token
- 边界框和轨迹：组嵌入仅注入到对应主体的空间区域和轨迹点

**作用**: 明确绑定机制，确保每个主体正确关联其对应的边界框和轨迹

#### 角色嵌入 (Role Embeddings)
- **对象嵌入 (Object Embedding)**: 添加到所有参考图像 token，指定为视觉外观资源
- **控制嵌入 (Control Embedding)**: 应用于所有边界框和轨迹 token，标记为运动控制指导

**作用**: 区分输入信号的功能，使模型能够有效处理异构条件

### 5. Latent Identity Reward Model (LIRM)

#### 架构设计

LIRM 包含三个关键模块：
1. **视频扩散模型 (VDM) 骨干**: 提取时空特征
2. **身份交叉注意力层**: 测量身份与视频内容的对齐
3. **奖励预测头**: 预测标量奖励

**工作流程**:
```
输入: 视频对 V ∈ {V_pos, V_neg} 和参考图像 I_ref
1. 通过 3D VAE 编码器投影到潜在空间: z_V, z_ref
2. 在时间步 t 扰动 z_V → z_V,t，保持 z_ref 干净
3. VDM 骨干提取特征:
   f_V = Φ(z_V,t, t, c_txt)
   f_ref = Φ(z_ref, t_0, c_txt)
4. 身份交叉注意力:
   h_attn = Attention(Q=f_ref W_Q, K=f_V W_K, V=f_V W_V)
5. 残差连接和 MLP 预测奖励:
   r_t = H(h_attn + Q)
```

#### 与传统方法的对比

**传统方法**:
- 使用静态图像编码器（CLIP、DINO）作为奖励模型
- 忽略时间动态
- 需要昂贵的 VAE 解码到像素空间
- 奖励反馈限制在最后的去噪步骤

**LIRM 的优势**:
1. **运动感知的身份评估**: 利用 VDM 的时空先验，评估视频级身份一致性
2. **计算高效**: 在潜在空间计算奖励，绕过 VAE 解码
3. **密集反馈**: 可在任意扩散时间步提供奖励反馈
4. **惩罚复制粘贴伪影**: 评估集成运动动态的身份一致性

#### 训练数据

- **训练集**: ~27,500 个视频
- **测试集**: 500 个视频
- 每个样本：视频输赢对 + 对应的单/多主体参考图像
- 人类注释标签：y ∈ {0, 1} 表示视频是否与参考图像定义的身份对齐

**损失函数**:
```
L_LIRM = -E_D_LIRM [y log σ(r_t) + (1-y) log (1-σ(r_t))]
```

### 6. Latent Identity Reward Feedback Learning (LIReFL)

#### 核心创新

标准 ReFL 在视频生成中面临严重计算瓶颈：
- 需要昂贵的 VAE 解码进行像素级评估
- 奖励反馈限制在最终去噪结果
- 忽略早期扩散阶段建立的结构信息

**LIReFL 的解决方案**:
- 绕过 VAE 解码器，显著减少内存开销
- 直接梯度反向传播到视频生成器
- 在任意扩散时间步提供密集奖励反馈

#### 训练流程

```
1. 从高斯噪声初始化潜在表示
2. 采样目标中间时间步 t_m ~ U(0, T-1)
3. 从步骤 T 到 t_m+1 执行标准无梯度去噪
4. 在步骤 t_m+1 执行单步梯度启用去噪:
   z_tm = μ_θ(z_{tm+1}, t_m+1, c_txt, z_ref)
5. 冻结的 LIRM 评估身份奖励:
   r_tm = LIRM(z_tm, t_m, c_txt, z_ref)
6. 强化损失（最大化身份保真度）:
   L_LIReFL = -E[ r_tm ]
```

#### 防止奖励黑客

为防止模型以牺牲视觉质量或多样性为代价过度优化身份分数，引入 SFT 损失作为正则化器：

```
L = L_sft + λ_2 * L_LIReFL
```

其中 λ_2 = 0.1 控制奖励反馈的强度

**平衡策略**确保：
- 与人类身份偏好对齐
- 保持 SFT 阶段建立的精确运动控制
- 保持生成多样性

## 与 Spatial AGI 的关系

### 直接贡献

#### 1. 统一的空间-时间控制

DreamVideo-Omni 的核心创新在于将多种空间和时间控制信号统一到单一框架中：

**空间控制**:
- 全局空间位置（边界框）
- 局部空间动态（点轨迹）
- 摄像机空间运动（背景轨迹）

**时间控制**:
- 时间一致性（3D RoPE）
- 运动时间演化（轨迹序列）

**对 Spatial AGI 的意义**: 提供了对空间场景的全面理解和控制能力，这是 Spatial AGI 的核心需求

#### 2. 多主体空间关系建模

**组和角色嵌入**:
- 明确区分不同主体
- 为每个主体建立独立的空间控制单元
- 解决多主体场景中的空间混淆

**对 Spatial AGI 的意义**: 能够理解和操作复杂多物体场景中的空间关系，这对于机器人操作、AR/VR 等应用至关重要

#### 3. 从静态到动态的空间理解

**LIRM 的运动感知评估**:
- 不再依赖静态图像编码器
- 利用 VDM 的时空先验
- 评估动态变化中的身份一致性

**对 Spatial AGI 的意义**: 从静态空间表示进化到动态空间理解，这是 Spatial AGI 从感知到行动的关键一步

## 实验设置和结果

### 实现细节

#### 基础模型
- **骨干**: Wan2.1-1.3B T2V
- **分辨率**: 480×832
- **帧数**: 49 帧
- **优化器**: AdamW

#### Stage 1 训练
- **迭代次数**: 40,000
- **GPU**: 64 NVIDIA A100
- **批大小**: 64
- **学习率**: 5×10^-5
- **权重衰减**: 1×10^-3
- **条件丢弃概率**: p = 0.5
- **数据增强概率**: p = 0.5
- **重加权扩散损失权重**: λ_1 = 2

#### Stage 2 训练

**LIRM 训练**:
- **GPU**: 16 A100
- **批大小**: 16
- **权重衰减**: 1×10^-2
- **步数**: ~4,000
- **学习率**: 
  - 预测头和注意力层: 1×10^-5
  - VDM 骨干: 1×10^-6
- **骨干**: Wan2.1-1.3B 的前 8 层

**LIReFL 训练**:
- **步数**: 3,400
- **学习率**: 5×10^-6
- **SFT 损失权重**: λ_2 = 0.1

#### 推理
- **调度器**: UniPC
- **步数**: 50
- **分类器自由引导尺度**: 5.0

### 数据集

#### 训练数据集

**规模**: 2M 视频片段

**注释类型**:
- 多主体参考图像
- 边界框
- 轨迹条件

**数据处理流水线**:

1. **运动过滤**
   - 工具: RAFT 光流
   - 标准: 平均运动幅度
   - 目的: 保留有意义运动模式

2. **主体发现和字幕**
   - 工具: RAM++ (语义标签)
   - 细化: Qwen3 Max
   - 字幕: Qwen3-VL

3. **时空注释提取**
   - 边界框: Grounding DINO
   - 掩码: SAM 2
   - 轨迹: CoTracker3
   - 分类: 主体掩码（物体 vs 摄像机）

4. **参考图像构建**
   - 从时间上不相交的帧采样
   - 通过掩码隔离主体
   - 广泛数据增强

#### DreamOmni Bench

**规模**: 1,027 视频
- 单主体: 436
- 多主体: 591

**类别**:
- 人类
- 一般物体
- 动物
- 面部

**特点**:
- 高质量真实世界视频
- 与训练数据集独立（零样本评估）
- 手动过滤（高分辨率、有意义运动、无水印）
- 密集注释（字幕、掩码、边界框、轨迹）

### 评估指标

#### 1. 整体一致性

**CLIP-Text Similarity (CLIP-T)**
- 模型: CLIP ViT-B/32
- 目的: 评估生成视频与文本提示的语义对齐

#### 2. 主体和面部保真度

**Region CLIP-Image Similarity (R-CLIP)**
- 检测: GroundingDINO
- 裁剪: 基于文本标签的主体区域
- 相似度: 裁剪区域与参考图像的余弦相似度
- 模型: CLIP ViT-B/32

**Region DINO-Image Similarity (R-DINO)**
- 模型: DINO-ViT-S/16
- 其他: 同 R-CLIP

**Face Similarity (Face-S)**
- 工具: InsightFace + ArcFace
- 流程: 
  1. 检测所有面部
  2. 提取嵌入
  3. 计算与参考面部的余弦相似度
  4. 匹配最高相似度的面部

#### 3. 运动控制精度

**Bounding Box Accuracy**
- 指标: IoU (Intersection over Union)
- 评估: 全局运动控制的准确性

**Trajectory Accuracy**
- 指标: 平均欧氏距离
- 评估: 局部运动和摄像机运动的精度

1. **条件感知 3D RoPE**
   - 移除后: 训练不稳定，收敛慢
   - 保留: 稳定训练，快速收敛

2. **组和角色嵌入**
   - 移除后: 多主体场景严重混淆
   - 保留: 清晰的主体区分

3. **分层运动注入**
   - 移除后: 全局运动控制不精确
   - 保留: 精确的全局控制

4. **LIRM**
   - 移除后: 大运动下身份退化
   - 保留: 健壮的身份保持

5. **LIReFL**
   - 移除后: 身份保真度降低
   - 保留: 与人类偏好对齐

#### 超参数敏感性

**条件丢弃概率 p**:
- p = 0: 过拟合，泛化差
- p = 0.5: 最佳平衡
- p = 1.0: 控制信号学习不足

**SFT 损失权重 λ_2**:
- λ_2 = 0: 无正则化，奖励黑客
- λ_2 = 0.1: 最佳平衡
- λ_2 = 1.0: 奖励学习不足

## 方法细节

### 模型架构

#### 输入处理

**视频输入**:
- 分辨率: 480×832
- 帧数: 49
- 编码: 3D VAE
- 潜在表示: z_t

**参考图像**:
- 每个主体一张
- 分割处理（空白背景）
- 编码: 3D VAE
- 潜在表示: z_ref

**边界框**:
- 渲染为 RGB 视频（白色背景）
- 每个物体唯一随机颜色
- 重叠区域像素平均
- 编码: 3D VAE
- 潜在表示: z_box

**轨迹**:
- 混合采样策略:
  - 随机网格采样: 全场景覆盖
  - 对象感知采样: 聚焦局部动态
- 构建轨迹 token:
  - 唯一正弦位置编码
  - 散射到空白特征图
- 潜在表示: z_traj

#### DiT 架构

**输入**:
- 噪声视频潜在: z_t
- 参考图像潜在: z_ref
- 边界框潜在: z_box
- 轨迹 token: z_traj

**处理流程**:
1. 沿时间维度连接所有潜在 token
2. 应用条件感知 3D RoPE
3. 分层注入边界框条件
4. DiT 块处理
5. 输出去噪预测

#### 条件注入

**主体外观**:
- 与噪声视频连接
- 组嵌入 + 对象嵌入

**全局运动**:
- 分层注入（输入 + 每个 DiT 块）
- 零卷积
- 组嵌入 + 控制嵌入

**局部运动/摄像机**:
- 与噪声视频连接
- 时间索引对齐
- 组嵌入 + 控制嵌入

### 训练策略

#### Stage 1 损失

**重加权扩散损失**:
```
L_sft = E[(1 + λ_1 M) · ||ε - ε_θ(z_t, C, t)||²_2]
```

其中:
- C = {c_txt, z_ref, z_box, z_traj}
- M: 二进制边界框掩码
- λ_1: 前景放大因子

**目的**:
- 放大边界框内区域的贡献
- 增强主体学习
- 保持背景区域原始损失

#### Stage 2 损失

**联合损失**:
```
L = L_sft + λ_2 * L_LIReFL
```

其中:
- L_sft: Stage 1 的监督损失
- L_LIReFL: 潜在身份奖励反馈损失
- λ_2: 平衡因子

**目的**:
- 与人类身份偏好对齐
- 保持运动控制和生成多样性
- 防止奖励黑客

### 数据增强

#### 参考图像增强

**几何变换**:
- 翻转
- 旋转
- 仿射剪切
- 裁剪

**视觉退化**:
- 模糊
- 颜色抖动

**目的**:
- 防止过拟合
- 减轻"复制粘贴"伪影
- 增强身份保持的鲁棒性

#### 条件丢弃

**策略**:
- 边界框: 50% 概率丢弃
- 轨迹: 50% 概率丢弃

**目的**:
- 增强鲁棒性
- 支持灵活的任务组合
- 防止条件依赖

## 技术细节补充

### 边界框处理

#### 过滤标准
- 检测相邻帧边界框 IoU
- 过滤掉具有突变物体波动的片段
- 目的: 确保训练稳定性

#### 渲染策略
- 白色背景
- 每个物体唯一随机颜色
- 重叠区域像素平均
- 目的: 解决模糊性

### 轨迹采样

#### 混合采样策略

**随机网格采样**:
- 覆盖全场景动态
- 包括背景和物体
- 确保广覆盖

**对象感知采样**:
- 严格在前景掩码内采样
- 聚焦复杂局部动态
- 捕获非刚性变形

#### 鲁棒性增强
- 训练期间随机丢弃轨迹子集
- 提高对轨迹密度的鲁棒性

### 位置编码

#### 轨迹 Token 构建
1. 生成唯一正弦位置编码
2. 根据离散化时空坐标散射到空白特征图
3. 与噪声视频和参考图像潜在连接

### LIRM 训练细节

#### 骨干选择
- 使用 VDM 的前 8 个块
- 原因: 减轻计算开销
- 参考: PRFL 方法

#### 冻结策略
- 冻结文本和补丁嵌入层
- 联合更新骨干、注意力层和预测头

#### 学习率策略
- 差分学习率
- 预测头和注意力层: 更高学习率
- VDM 骨干: 更低学习率

### LIReFL 训练细节

#### 单步梯度去噪
- 从步骤 T 到 t_m+1: 无梯度去噪
- 在步骤 t_m+1: 单步梯度启用去噪
- 目的: 节省内存

#### 中间时间步采样
- t_m ~ U(0, T-1)
- 均匀采样整个去噪过程
- 提供密集反馈

4. **潜在空间优化**: 为高效的强化学习提供了新思路
