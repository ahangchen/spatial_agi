## 幻灯片3：核心创新 - VST范式 (45秒)

**Video Streaming Thinking (VST)**

💡 **关键思想**：边看边思考 (Thinking While Watching)

**对比传统方法**：
```
传统：Query → Heavy Reasoning → Answer (8.8s)
VST：  Stream + Light Reasoning → Query → Answer (0.56s)
```

**数学原理**：
```
p(y|q,V) = p(y|q,c^K,m^K) × ∏_{k=1}^{K-1} p(z^k|c^k,m^{k-1})
           └─ Direct Answer─┘   └─── Streaming Thinking ───┘
```

**核心优势**：
- ✅ 推理延迟被"分摊"到视频播放期间
- ✅ 15.7倍响应加速
- ✅ 保持强推理能力

## 幻灯片4：技术架构 - 双记忆系统 (45秒)

**双记忆协同机制**

📦 **短期视觉记忆** (Visual Buffer)
- 大小：8192 tokens (约4秒视频)
- 内容：当前视觉细节
- 作用：提供实时感知

📚 **长期文本记忆** (Textual Memory)
- 形式：文本序列
- 内容：累积的历史事件和语义
- 管理：FIFO自动淘汰

**工作流程**：
```
视频片段 c^k + 历史记忆 m^{k-1}
    ↓
生成流式思考 z^k
    ↓
更新记忆 m^k = FIFO(m^{k-1}, z^k)
```

## 幻灯片5：训练流程 (45秒)

**两阶段后训练**

**阶段1：VST-SFT** (监督微调)
- 目标：学习因果时序推理
- 方法：流式注意力掩码 + 时间分段
- 效果：Backward能力 +9.2%

**阶段2：VST-RL** (强化学习)
- 目标：端到端优化推理质量
- 方法：GRPO算法，稀疏奖励
- 效果：Forward能力 +12.7%

**数据合成**：
- 基于知识图谱的自动化流程
- 100K高质量流式推理样本
- 实体-关系基础的CoT生成

## 幻灯片10：局限与未来 (30秒)

**当前局限**
- 计算开销增加（总token数增加）
- 文本记忆可能丢失视觉细节
- 固定思考频率可能冗余

**未来方向**
- 🔮 潜在推理：token高效的推理方式
- 🎯 自适应思考：根据场景动态调整
- 🌈 多模态记忆：视觉+文本混合表示
- 🔗 与视觉记忆结合：协同而非替代
