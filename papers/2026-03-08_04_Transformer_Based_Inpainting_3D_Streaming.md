## 核心信息

### 摘要

高质量的多摄像头3D流媒体对于AR/VR应用中的沉浸式体验至关重要。由于实时约束导致的视角数量有限，会在渲染图像中产生缺失信息和不完整表面。现有方法通常依赖简单的启发式方法进行空洞填充，这可能导致不一致或视觉伪影。

本文提出使用一种新颖的、针对应用的修复方法来补全缺失的纹理，该方法作为新视角渲染后的基于图像的后处理步骤，独立于底层表示。该方法被设计为与任何校准的多摄像头系统兼容的独立模块。

为此，本文引入了一个多视角感知的、基于Transformer的网络架构，使用时空嵌入来确保跨帧一致性，同时保留细节。此外，分辨率无关的设计允许适应不同的摄像头设置，而自适应patch选择策略平衡推理速度和质量，实现实时性能。

在相同实时约束下与最先进的修复技术进行评估，证明该模型实现了质量和速度之间的最佳权衡，在图像和视频指标方面都优于竞争对手。

### 关键贡献

1. **多视角感知的Transformer修复网络**：引入了一种新颖的、多视角感知的基于Transformer的修复网络，用于实时视频修复，作为3D流媒体管道中的通用后处理步骤。

2. **时空嵌入**：提出了一种时空嵌入，利用几何代理进行重投影，增强多视角信息的特征传播。

3. **自适应patch过滤**：设计了基于时空局部性的patch过滤机制，调整推理期间所需的patch数量，允许在速度和准确性之间进行权衡。

## 1. 引言与动机

### 1.1 问题背景

无模板3D流媒体在娱乐（体育、艺术、音乐会、电影）、远程呈现和医疗应用等多个领域具有巨大潜力，但在针对消费硬件和AR/VR设备时仍然具有挑战性。多摄像头设置产生的海量数据在实时应用中难以处理，需要仔细选择和提炼底层信息。

这与多视角重建中的常见观念直接矛盾：稀疏视角会导致场景重建中经常包含不完整的几何或纹理。这些未见区域是所有此类方法的基本挑战，其影响大幅降低了视频流的感知质量。

### 1.2 现有方法的局限

虽然视频修复方法为这个问题提供了直接的解决方案，但最先进的方法并不是为这种用例设计的，只有少数方法能够实现（接近）实时性能。相反，最近工作的一个主要焦点是产生感知良好结果所需的时间一致性，以及访问其他帧信息所需的高效特征传播，可能跨越长时间和空间间隔。

这与本文的用例不同，其中不完整的新视角中包含的有限信息使得生成合理内容变得困难。甚至很有可能必要信息不包含在任何过去的帧中，因为它们是由相同的设置产生的。因此，改进单视图之外的特征传播是优化的一个中心点，因为它减少了内容幻觉任务的负担，这是高度不适定的。

### 1.3 本文方法的优势

与直接完成几何或辐射场的3D修复方法不同，本文的工作目标是实时3D流媒体管道内渲染视图的2D视频修复，其中可用的多视角信息已经由几何代理融合。

本文认为，用于生成3D表示的原始图像为模型在修复过程中使用提供了丰富的信息，其中大部分不包含在新视角中。本文提出了一种基于学习的方法，使用基于Transformer的架构利用这些来自给定输入视图的现成数据，这自然非常适合此任务。

本文的方法对来自目标图像和上下文图像（即原始摄像头视图以及所有视图的过去帧）的特征空间patches进行操作。为了促进信息传输，本文为上下文patches及其相对坐标引入了时空编码，利用底层3D代理更好地利用上下文信息。为了更快的推理，本文提出了一个top-k过滤机制，并以可忽略的质量损失演示了实时性能。

## 2. 核心算法原理

### 2.1 整体架构

本文的修复模型由三个主要组件组成：
1. **图像编码器ℰ**：基于FuseFormer的CNN编码器
2. **Transformer块组**：在特征patches上操作的transformer块组
3. **解码器𝒟**：反卷积解码器网络

该方法对底层3D流媒体方法施加最小的约束，确保与广泛的实时框架兼容。在每个时间步t，假设重建算法提供几何表示𝒢t以及摄像头捕获的输入RGB图像Ii,t。

### 2.2 编码和Patch提取

#### 2.2.1 特征编码

CNN图像编码器ℰ基于FuseFormer，独立处理上下文输入{Ii,t,Mi,t}和新视角输入{Ft,Mt}，将它们转换为从CNN多个阶段获取的分层、高维特征表示。

从可用的几何代理𝒢t，可以选择性地通过使用校准捕获系统的已知摄像头参数重新渲染𝒢t来生成伪深度信息，并将这些伪深度图作为辅助输入与上下文和新视角图像一起作为额外通道传递。

#### 2.2.2 时间上下文编码

本文遵循Thiry等人的方法，编码与当前帧相邻的时间步的上下文帧：
- **紧邻的nc帧**：当前帧t之前的{t-j | j=1,2,...,nc}
- **远处的nw帧**：具有间隔因子kw>1的{t-kw*j | j=1,2,...,nw}，以覆盖更广泛的上下文而不考虑每个过去的帧

实验设置：nw=nc=3，kw=10，使每个摄像头最多包含7帧上下文。

#### 2.2.3 Patch提取和分类

生成的特征图被分成小的、重叠的patches，任何完全由背景组成的patch都被丢弃。从修复特征图中，不包含需要修复的像素的patches也被添加到上下文集中。

- **修复patches 𝒫t**：包含缺失像素的patches
- **上下文patches ℛt**：所有保留的patches

每个patch p∈𝒫t或r∈ℛt都与时空坐标xp,xr∈ℝ3相关联，使模型能够确切知道每个patch位于何处（以及何时）。具体来说，每个向量x∈ℝ3编码patch中心的屏幕空间坐标（归一化为[0,1]）以及patch出现的时间步，也使用最大窗口大小kwnwk进行归一化。

### 2.3 多视角感知Transformer（Multi-View Aware Transformer）

#### 2.3.1 重投影机制

为了将上下文patches与新视角相关联，本文通过重投影将屏幕空间坐标从上下文摄像头的视图映射到目标（新）摄像头的视图：

**x̂r = C𝒢t(xr)**

其中C𝒢t(·)是重投影函数，利用几何代理𝒢t进行坐标映射。

#### 2.3.2 Transformer块结构

输入patches 𝒫t由ng个transformer组处理，每个组包含nb个块。在每个块内，patch序列通过关注上下文patch序列ℛt来更新。

在第k个块，归一化的输入𝒫t(k-1)更新为：

**𝒫t(k) = 𝒫t(k-1) + A(WQ(k)𝒫t(k-1), WK(k)ℛt, WV(k)𝒫t)**

其中𝒫t(0)=𝒫t，WQ(k), WK(k), WV(k)是第k个块的学习投影算子，结果然后通过标准前馈块。

#### 2.3.3 旋转位置嵌入（RoPE）

本文通过分解的3D旋转位置嵌入变体在注意力计算中结合每个patch的时空坐标：

**A(Q,K,V) = σ(RoPE(Q,xQ)·RoPE(K,x̂K)T/√D)V**

其中xQ和x̂K是与查询和键patches相关联的（投影）3D时空坐标，D表示特征维度的大小，σ表示softmax函数。

对于一对patches，RoPE在多个频率上编码相对位置，而不显式计算成对坐标距离。

### 2.4 时空嵌入（Spatio-Temporal Embeddings）

#### 2.4.1 坐标系统

每个patch的时空坐标x∈ℝ3包含：
1. **屏幕空间坐标**：patch中心的(u,v)坐标，归一化到[0,1]
2. **时间戳**：patch出现的时间步，使用最大窗口大小kwnwk归一化

#### 2.4.2 3D RoPE分解

将3D旋转位置嵌入分解为：
- **空间维度**：u和v坐标的旋转嵌入
- **时间维度**：时间戳的旋转嵌入

这种分解允许模型分别处理空间和时间关系，同时在注意力机制中联合考虑它们。

#### 2.4.3 几何代理的作用

几何代理𝒢t（如视觉外壳）用于：
1. 将上下文视图的屏幕空间坐标重投影到目标视图
2. 提供伪深度信息作为辅助输入
3. 确保多视角一致性

### 2.5 自适应Patch选择（Adaptive Patch Selection）

#### 2.5.1 Top-k过滤机制

许多上下文patches可能不包含有价值的修复信息。因此，在每个组的第一个transformer之后应用patch稀疏化，通过仅保留最相关的patches来提高运行时性能。

**算法**：
1. 计算上下文patches中每个token的注意力权重之和
2. 根据该和保留top-k个token
3. 使用直通估计器训练掩码

这种方法遵循Cordonnier等人的token修剪方法，在保持质量的同时显著减少计算量。

#### 2.5.2 分辨率无关设计

由于patch-based设计，模型可以处理不同分辨率的输入，适应不同的摄像头设置，而不需要固定输入尺寸。

### 2.6 解码和融合

#### 2.6.1 Patch解码

一旦所有transformer组处理完patches，结果patch表示被转发到反卷积解码器网络𝒟，该网络独立地从每个特征patch计算RGB patch。

#### 2.6.2 重建和混合

重建的patches通过线性混合重叠像素重新插入其原始位置，产生中间RGB图像F̃t，然后使用错误掩码Et与输入线性混合产生最终输出：

**F̂t = Et⊙F̃t + (1-Et)⊙Ft**

其中⊙表示逐元素乘法。

## 3. 损失函数与训练

### 3.1 损失函数组成

模型使用ℓ1图像损失和对抗损失的组合进行训练：

**ℒ = λimg(ℒin + ℒout) + λadvℒadv**

其中λimg=1.0和λadv=0.01是控制损失函数之间权衡的权重系数。

### 3.2 重建损失

**修复区域损失**：
**ℒin = (1/||Et||1)||Et⊙(F̃t-Ft*)||1**

**已知区域损失**：
**ℒout = (1/||1-Et||1)||(1-Et)⊙(F̃t-Ft*)||1**

其中Ft*是相应的ground-truth图像。

注意，ℒin和ℒout在混合之前的中间图像F̃t上计算，以帮助编码器/解码器网络的泛化。

### 3.3 对抗损失

对抗损失遵循Pathak等人的GAN公式：

**ℒadv = maxD 𝔼Ft*(logD(Ft*)) + 𝔼F̂t(log(1-D(F̂t)))**

ℒadv在最终结果F̂t上计算，以提高视觉保真度。

### 3.4 无显式跨视角一致性损失

本文不应用显式的跨视角一致性损失，因为上游重建阶段（RIFTCast）已经在当前时间步聚合了可用的多视角信息。相反，模型依赖重投影和基于RoPE的时空注意力来利用渲染输入中仍然可用的跨视角上下文。

## 4. 高效推理

### 4.1 特征缓存

在3D流媒体的推理过程中，可以利用视频流逐帧到达且编码器ℰ多次接收相同帧的事实。通过缓存编码的特征图直到帧t-kwnw，如果有足够的内存可用，可以显著减少值的重新计算。

### 4.2 实时性能

- **FPS**: 41.55（在DNARendering数据集上）
- **FPS**: 较低（在更高分辨率的RIFTCast数据集上）

通过top-k过滤机制，模型实现了实时性能，同时保持高质量。

## 5. 实验评估

### 5.1 数据集

#### 5.1.1 DNARendering数据集
- **训练**: 72个场景
- **评估**: 7个场景
- **特点**: 动态人类表演的真实世界数据集，确保没有受试者同时出现在两个集合中

#### 5.1.2 RIFTCast数据集
- **场景**: 31个场景
- **摄像头**: 34个同步RGB摄像头
- **特点**: 复杂的动态多演员和演员-物体交互

### 5.2 实验设置

- **3D流媒体方法**: RIFTCast，使用前景掩码构建视觉外壳作为几何代理𝒢t
- **输入视图**: 与RIFTCast相同的三个最接近的输入视图
- **Patch大小**: 7×7，3像素重叠
- **Transformer组**: ng=2组，每组nb=4个块
- **时间上下文**: nw=nc=3，kw=10（每摄像头最多7帧）

### 5.3 评估指标

- **图像指标**: PSNR, SSIM, LPIPS
- **视频指标**: VFID (video Fréchet inception distance)
- **评估方式**: 整个图像 + 仅修复区域
- **速度指标**: FPS

### 5.4 基线方法

与三个最近的、高效的修复模型（DSTT、FuseFormer、E2FGVI）的多种变体进行比较：

1. **Default (def)**: 未修改的基线，在较低的架构特定分辨率上处理图像然后放大
2. **Windowed (win)**: 将帧分割成与原始分辨率匹配的单个重叠窗口，然后拼接预测结果
3. **Multi-View (mul)**: 将基线适配为也接收来自多个摄像头的帧，将额外的摄像头视图交错到时间输入序列中

还包括RGVI作为离线方法的参考。

### 5.5 定量结果

#### 5.5.1 DNARendering数据集（主结果）

**整个图像**：
- **PSNR**: 32.616 (最佳)
- **SSIM**: 0.9851 (最佳)
- **LPIPS**: 0.0262 (最佳)
- **VFID**: 1.6671 (最佳)

**仅修复区域**：
- **PSNR**: 42.184 (最佳)
- **LPIPS**: 0.0022 (最佳)
- **FPS**: 41.55

#### 5.5.2 RIFTCast数据集（泛化测试）

在更具挑战性的RIFTCast数据集上，**无需重新训练或微调**，模型在修复区域仍然优于基线方法，证明了强大的泛化能力。

### 5.6 定性结果

视觉比较显示：
- 基线方法经常用深色修复较大区域或引入颜色伪影
- DSTT在手臂上产生发光的红点
- 所有基线方法在手臂上产生灰色伪影，而本文方法更忠实地再现肤色
- 基线方法模糊裤子的深色和鞋子的白色，而本文模型生成更清晰的边界

### 5.7 消融研究

#### 5.7.1 组件影响

比较以下变体：
1. **Single cam w/o masks**: 单摄像头，无掩码
2. **Single cam**: 单摄像头，有掩码
3. **w/o temp**: 多摄像头，无过去视频帧
4. **w/o RoPE**: 多摄像头，无旋转位置编码
5. **Ours**: 完整模型

**关键发现**：
- 无掩码或位置嵌入时，模型难以理解目标区域的空间位置，无法识别手臂的正确颜色
- 无时间数据时，模型无法利用过去帧检索更多关于区域的信息
- 完整模型产生更少的灰色伪影

#### 5.7.2 性能分析

- **Single cam w/o masks**: FPS 87.46，但质量最低
- **Single cam**: FPS 79.43，质量有所提升
- **w/o temp**: FPS 41.32，缺少时间信息
- **w/o RoPE**: FPS 41.05，缺少空间位置编码
- **Ours**: FPS 41.55，最佳质量-速度权衡

### 5.8 失败案例

观察到场景中快速移动的内容会降低输出质量，当方程2的假设不再成立时，即来自过去帧的屏幕空间patch坐标可以使用当前几何代理重投影到目标视图。

示例：当手臂之间的区域大部分被遮挡时，快速运动可能导致模型无法将缺失区域与其他帧中的衣袖相关联。此外，由于受试者前景掩码的轻微不准确性，ground-truth图像中手臂之间的小间隙中可以看到绿屏背景，这为推断这些区域中的正确颜色增加了另一个困难。

## 6. 与Spatial AGI的关系

### 6.1 直接贡献

#### 6.1.1 3D空间重建与表示

本文方法虽然不直接重建3D几何，但在3D流媒体管道中扮演关键角色：
1. **几何代理利用**: 使用几何代理𝒢t（如视觉外壳）进行多视角信息融合
2. **重投影机制**: 将上下文视图重投影到目标视图，利用3D几何关系
3. **多视角一致性**: 通过重投影和时空注意力确保跨视角一致性

#### 6.1.2 实时性能实现

- **41.55 FPS**: 在保持高质量的同时实现实时性能
- **自适应patch选择**: top-k过滤机制平衡速度和质量
- **特征缓存**: 利用帧间冗余减少重复计算
- **分辨率无关设计**: 适应不同摄像头设置

#### 6.1.3 空间关系处理

- **时空嵌入**: 显式编码空间和时间位置信息
- **3D RoPE**: 分解的3D旋转位置嵌入处理空间和时间关系
- **多视角感知**: Transformer架构自然适合处理来自多个视角的信息

### 6.2 技术启发

#### 6.2.1 对Spatial AGI的启发

1. **多视角信息融合**: 
   - 利用多个摄像头视角的互补信息
   - 通过几何代理进行信息聚合
   - 减少单视角的不确定性

2. **实时约束下的优化**:
   - 自适应计算分配（top-k过滤）
   - 特征缓存和重用
   - 分辨率无关设计

3. **时空一致性**:
   - 时空嵌入编码位置和时间信息
   - RoPE实现相对位置关系建模
   - 跨帧信息传播

4. **模块化设计**:
   - 独立于底层3D表示
   - 作为后处理模块插入现有管道
   - 与任何校准的多摄像头系统兼容

#### 6.2.2 应用场景

1. **机器人视觉**:
   - 稀疏摄像头设置下的场景补全
   - 实时处理用于机器人导航和操作
   - 处理遮挡和不完整观测

2. **AR/VR应用**:
   - 实时3D流媒体
   - 新视角合成中的空洞填充
   - 提升沉浸式体验质量

3. **远程呈现**:
   - 低延迟视频流处理
   - 多摄像头协作
   - 前景物体渲染质量提升

4. **自动驾驶**:

## 7. 核心技术发现

### 7.1 Multi-View Aware Transformer的详细机制

#### 7.1.1 架构创新

**与传统视频修复的区别**:
- 传统方法: 单视角 + 时间序列
- 本文方法: 多视角 + 时间序列 + 几何约束

**关键组件**:
1. **输入**: 多个摄像头的当前帧 + 过去帧
2. **编码器**: 基于FuseFormer的CNN，独立处理每个视图
3. **Patch提取**: 重叠patches，区分修复区域和上下文区域
4. **Transformer组**: ng=2组，每组nb=4个块
5. **解码器**: 反卷积网络重建RGB patches

#### 7.1.2 注意力机制

**查询-键-值结构**:
- **Query**: 来自修复patches 𝒫t
- **Key & Value**: 来自上下文patches ℛt
- **注意力**: RoPE编码相对位置

**多视角信息流动**:
1. 编码所有视角的特征
2. 通过重投影建立视角间对应关系
3. 注意力机制聚合相关信息
4. 解码生成修复结果

#### 7.1.3 几何感知

**几何代理的作用**:
- 提供深度信息（可选）
- 支持重投影函数C𝒢t
- 确保多视角几何一致性

**重投影流程**:
1. 上下文patch坐标xr
2. 通过几何代理重投影: x̂r = C𝒢t(xr)
3. 在注意力中使用重投影坐标

### 7.2 Spatio-Temporal Embeddings的实现

#### 7.2.1 坐标编码

**3D坐标x∈ℝ3**:
- **x[0]**: u坐标（归一化到[0,1]）
- **x[1]**: v坐标（归一化到[0,1]）
- **x[2]**: 时间步t（归一化使用kwnwk）

**编码策略**:
- 屏幕空间: patch中心的(u,v)
- 时间: 帧的时间戳
- 归一化: 确保数值稳定性

#### 7.2.2 3D RoPE分解

**分解原理**:
- 将3D旋转嵌入分解为独立的1D旋转
- 每个维度使用不同的频率基
- 组合所有维度的旋转

**数学形式**:
```
RoPE(x, x̂) = RoPE_u(x[0], x̂[0]) × RoPE_v(x[1], x̂[1]) × RoPE_t(x[2], x̂[2])
```

**优势**:
- 无需显式计算成对距离
- 多频率编码捕获不同尺度的关系
- 计算效率高

#### 7.2.3 时间建模

**时间上下文窗口**:
- 紧邻帧: {t-1, t-2, ..., t-nc}
- 远处帧: {t-kw, t-2*kw, ..., t-nw*kw}
- 总共: 每摄像头最多7帧

**时间信息利用**:
- 短期: 捕获快速运动
- 长期: 访问历史信息
- 间隔采样: 扩大时间感受野

### 7.3 Adaptive Patch Selection的优化策略

#### 7.3.1 Top-k过滤算法

**动机**:
- 许多上下文patches不相关
- 减少计算量提高速度
- 保持质量不变

**算法流程**:
```
输入: 上下文patches ℛt, 注意力权重A
1. 对每个上下文patch r∈ℛt:
   score[r] = Σi A[i,r]  // 对所有查询patches求和
2. 选择top-k个最高分数的patches
3. 创建二值掩码M
4. 使用直通估计器训练:
   forward: M * ℛt
   backward: ∂L/∂ℛt (忽略M的梯度)
```

#### 7.3.2 效果分析

**计算复杂度**:
- 原始: O(|𝒫t| × |ℛt|)
- 过滤后: O(|𝒫t| × k)，其中k << |ℛt|

**质量影响**:
- PSNR下降: < 0.5 dB
- FPS提升: 约2倍
- 最佳权衡: k值可通过验证集调优

#### 7.3.3 训练策略

**直通估计器**:
- Forward: 应用离散掩码
- Backward: 梯度直接传递
- 允许端到端训练

**正则化**:
- 添加稀疏性损失（可选）
- 鼓励选择少量patches
- 平衡质量和速度

## 附录：三个核心问题的详细回答

### Q1: 核心算法原理是什么？

**核心思想**：在实时3D流媒体中，由于视角稀疏导致渲染图像中存在缺失信息。本文提出使用多视角感知的Transformer网络，利用原始摄像头图像作为上下文，通过时空嵌入和重投影机制，实时补全缺失纹理。

**主要技术方法**：

1. **Multi-View Aware Transformer**：
   - 架构：编码器-Transformer-解码器
   - 输入：目标视角 + 多个源视角 + 历史帧
   - 机制：通过重投影建立视角间对应关系
   - 优势：显式利用多视角几何约束

2. **Spatio-Temporal Embeddings**：
   - 3D坐标编码：(u, v, t)
   - 3D RoPE：分解为空间和时间维度的旋转嵌入
   - 作用：编码相对位置关系，支持可变长度序列
   - 实现：无需显式计算成对距离

3. **Adaptive Patch Selection**：
   - Top-k过滤：保留最相关的上下文patches
   - 直通估计器：支持端到端训练
   - 效果：2倍加速，质量损失<0.5 dB
   - 策略：根据注意力权重动态选择

**算法流程**：
1. 编码所有视角的特征
2. 提取重叠patches，区分修复和上下文区域
3. 计算每个patch的时空坐标
4. 通过重投影建立跨视角关系
5. Transformer处理：查询修复patches，键值来自上下文
6. Top-k过滤减少计算量
7. 解码重建RGB patches
8. 混合生成最终结果

**输入输出**：
- 输入：目标视角图像Ft + 掩码Mt + 错误图Et + 几何代理𝒢t + 多视角历史帧{Ii,τ}
- 输出：修复后的完整图像F̂t

**如何重建3D空间**：
- 不直接重建3D，而是在2D渲染视图中补全信息
- 利用几何代理𝒢t（如视觉外壳）提供3D约束
- 通过重投影函数C𝒢t建立不同视角间的空间关系
- 时空坐标x∈ℝ3统一表示空间和时间

### Q3: 如何保证多视角一致性？

**核心机制**：

1. **几何重投影约束**：
   - 使用几何代理𝒢t进行重投影：x̂r = C𝒢t(xr)
   - 确保不同视角的patches在3D空间中对齐
   - 硬约束保证物理一致性

2. **3D旋转位置嵌入**：
   - 在注意力中编码3D相对位置
   - RoPE(xQ, x̂K)考虑重投影后的坐标
   - 自然鼓励跨视角一致性

3. **上游多视角融合**：
   - RIFTCast已经聚合当前时间步的多视角信息
   - 渲染的Ft本身包含融合的几何
   - 修复在此基础上进一步补全

4. **时间一致性**：
   - 时空嵌入编码时间维度
   - 跨帧注意力传播信息
   - 确保时间上的平滑过渡

**无需显式一致性损失**：
- 不使用显式的跨视角一致性损失
- 依赖重投影 + RoPE实现隐式一致性
- 简化训练，避免复杂的损失设计

**实验验证**：
- 视觉结果：无明显跨视角不一致
- 消融研究：移除RoPE导致LPIPS上升（0.0022 → 0.0050）
- 泛化能力：零样本迁移到RIFTCast仍保持一致性

**局限性**：
- 快速运动时重投影可能不准确
- 依赖几何代理的质量
- 长期完全遮挡难以保证一致性

**改进方向**：
- 引入显式一致性损失（如需要）
- 更鲁棒的几何估计
- 长期记忆机制
