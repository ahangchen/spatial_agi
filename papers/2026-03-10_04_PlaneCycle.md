## 摘要

大规模2D基础模型展现了强大的可迁移表征能力,但将其扩展到3D体积数据通常需要重新训练、适配器(adapter)或架构重新设计。本文提出了PlaneCycle,一种无需训练、无需适配器的算子,用于架构无关的2D到3D提升。

PlaneCycle通过在网络深度方向上循环地将空间聚合分布在正交的HW、DW和DH平面上,复用原始预训练的2D骨干网络,实现了渐进式3D融合,同时保留了预训练的归纳偏置。该方法不引入额外的参数,适用于任意的2D网络。

使用预训练的DINOv3模型,研究者在六个3D分类和三个3D分割基准上评估了PlaneCycle。在无需任何训练的情况下,提升后的模型展现了内在的3D融合能力;在线性探测(linear probing)下,其性能超越了逐切片的2D基线和强3D对比方法,接近完全训练模型的性能。在全微调下,PlaneCycle匹配了标准3D架构的性能,突显了其作为无缝且实用的2D到3D提升算子的潜力。

这些结果证明了3D能力可以从预训练的2D基础模型中解锁,而无需结构修改或重新训练。

## 1. 引言与动机

### 1.1 研究背景

近年来,2D基础模型(如Vision Transformer、CLIP、DINO等)在大规模图像数据上的预训练取得了巨大成功,展现了强大的表征学习和迁移能力。这些模型通过在海量数据上学习,获得了丰富的视觉知识和强大的归纳偏置。

然而,将这些2D模型扩展到3D体积数据(如医学影像CT/MRI、3D物体识别等)面临着重大挑战:

1. **维度不匹配**: 2D模型设计用于处理平面图像,无法直接处理3D体积数据
2. **计算复杂度**: 3D卷积或3D Transformer的计算成本远高于2D对应物
3. **数据稀缺**: 3D标注数据相对稀少,难以支持大规模预训练
4. **架构限制**: 现有的2D模型架构难以直接适配3D任务

### 1.2 现有方法的局限

当前主流的2D到3D迁移方法包括:

#### 1.2.1 逐切片推理(Slice-wise Inference)
- **方法**: 将3D体积切片为多个2D图像,使用2D模型逐个处理
- **优点**: 简单直接,无需修改模型
- **缺点**:
  - 切片间信息隔离,缺乏3D上下文
  - 特征在深度方向不连贯
  - 无法捕捉完整的3D空间依赖

#### 1.2.2 适配器方法(Adapter-based Methods)
- **方法**: 在2D模型中插入可学习的适配器模块
- **优点**: 保留预训练权重,引入少量参数
- **缺点**:
  - 需要额外的训练
  - 增加了模型复杂度
  - 可能破坏预训练的表征能力

#### 1.2.3 3D架构重新设计
- **方法**: 设计专门的3D网络架构(如3D CNN、3D ViT)
- **优点**: 针对3D任务优化
- **缺点**:
  - 无法利用2D预训练权重
  - 需要大规模3D数据重新训练
  - 计算成本高昂

### 1.3 PlaneCycle的创新点

PlaneCycle提出了一种全新的范式:

1. **Training-Free**: 完全不需要训练,直接使用预训练权重
2. **Adapter-Free**: 不引入任何额外的可学习参数
3. **Architecture-Agnostic**: 适用于任意的2D网络架构
4. **Inductive Bias Preservation**: 完整保留预训练模型的归纳偏置

通过巧妙的架构设计,PlaneCycle证明了3D表征能力可以从2D基础模型中被"解锁",而无需任何结构修改或重新训练。

## 2. 核心算法原理

### 2.1 概述

PlaneCycle是一种旨在将2D基础模型(如ViT或CNN)提升至3D容积数据处理的**无须训练、无须适配器(Adapter-free)**的算子。其核心原理是通过在网络深度方向上,循环地将空间聚合分布在正交的二维平面上,从而解锁预训练2D模型的3D表征能力。

### 2.2 无需Adapter的2D-to-3D提升方法

PlaneCycle实现了一种**架构无关(Architecture-agnostic)**的提升方式,直接复用原始预训练2D骨干网络的权重,而不需要引入额外的参数或适配器。

#### 2.2.1 核心思想

给定一个3D特征图 $X \in R^{D \times H \times W \times C}$,PlaneCycle的核心思想是:

1. **重塑(Reshape)**: 根据选择的轴将容积数据重塑为P个切片(Slices)
2. **展平(Flatten)**: 将每个切片展平为令牌(Token)序列
3. **处理(Process)**: 切片序列独立地通过预训练的2D层进行特征聚合
4. **还原(Restore)**: 将Token重新映射回3D布局,保持输入输出的结构一致性

#### 2.2.2 权重复用机制

- **冻结参数**: 保持骨干网络参数完全冻结,不进行任何更新
- **直接推理**: 将3D数据的2D切片直接输入预训练层
- **批量处理**: 所有切片共享相同的权重,实现高效的批量计算

#### 2.2.3 架构无关性

PlaneCycle适用于各种2D架构:

- **Transformer-based**: 如ViT、DeiT、DINO等
  - 利用自注意力机制进行空间聚合
  - 保持位置编码的有效性

- **CNN-based**: 如ResNet、ConvNeXt等
  - 利用卷积操作进行局部特征提取
  - 保持平移不变性

### 2.3 HW、DW、DH平面的循环分布

为了捕捉完整的3D空间依赖关系,PlaneCycle在网络的不同层中交替改变聚合平面的方向。

#### 2.3.1 正交平面定义

PlaneCycle涵盖三个正交平面:

1. **HW平面(Axial/轴向)**
   - 沿深度轴D切片
   - 形状: $H \times W$
   - 对应医学影像中的横断面
   - 通常具有最高的分辨率和最强的解剖连续性

2. **DW平面(Coronal/冠状)**
   - 沿高度轴H切片
   - 形状: $D \times W$
   - 对应医学影像中的冠状面
   - 捕捉前后方向的空间关系

3. **DH平面(Sagittal/矢状)**
   - 沿宽度轴W切片
   - 形状: $D \times H$
   - 对应医学影像中的矢状面
   - 捕捉左右方向的空间关系

#### 2.3.2 循环调度策略

在实际应用中,PlaneCycle采用一个**四算子循环**:

$$
\text{Cycle}: HW \to DW \to DH \to HW
$$

**设计原理**:

1. **渐进式融合**: 随着网络深度增加,从不同视角观察3D体积
2. **信息交织**: 每一层处理不同平面,信息在正交方向上交织融合
3. **偏置分配**: 重复HW平面是因为:
   - 医学3D数据在轴向上通常具有更高分辨率
   - 轴向切片具有更强的解剖连续性
   - 分配更多的计算容量到信息最丰富的平面

#### 2.3.3 实现细节

```python
# 伪代码示例
def planecycle_forward(x_3d, layers):
    """
    x_3d: [D, H, W, C] - 3D特征图
    layers: 网络层列表
    """
    planes = ['HW', 'DW', 'DH', 'HW']  # 循环顺序

    for i, layer in enumerate(layers):
        plane = planes[i % len(planes)]

        # 根据平面重塑数据
        if plane == 'HW':
            slices = reshape_hw(x_3d)  # [D, H*W, C]
        elif plane == 'DW':
            slices = reshape_dw(x_3d)  # [H, D*W, C]
        else:  # DH
            slices = reshape_dh(x_3d)  # [W, D*H, C]

        # 通过2D层处理
        processed = layer(slices)

        # 还原为3D布局
        x_3d = restore_3d(processed, plane)

    return x_3d
```

### 2.4 3D融合的渐进式过程

通过在各层之间循环切换聚合平面,PlaneCycle实现了**渐进式的3D集成**。

#### 2.4.1 特征对齐机制

**问题**: 在零训练(Zero-training)的情况下,单平面的2D推理会导致跨切片特征的不一致

**解决方案**: PlaneCycle通过多平面循环实现特征对齐

- **一致性约束**: 特征被迫在三个正交平面上寻找共同的结构化规律
- **结构化表征**: 产生的特征在三个正交平面上都能表现出良好的对齐性
- **内在融合**: 即使没有额外的3D监督信号,也能学习到全局连贯的3D表征

#### 2.4.2 信息渗透过程

随着特征流经网络的深度,信息在正交方向上不断交织融合:

1. **浅层**:
   - 每个平面独立提取2D特征
   - 捕捉局部纹理和边缘信息

2. **中层**:
   - 不同平面的信息开始融合
   - 建立跨维度的空间关系

3. **深层**:
   - 形成全局连贯的3D表征
   - 整合所有平面的信息

#### 2.4.3 实验验证

实验表明,PlaneCycle产生的特征图(Feature Maps)在3D空间中具有很强的判别力和结构化的一致性,即使不进行任何训练。

### 2.5 如何保留预训练的归纳偏置

PlaneCycle的关键优势在于它能够完整地保留大规模2D预训练中获得的强大表征和**归纳偏置(Inductive Biases)**。

#### 2.5.1 参数冻结策略

**核心理念**: 参数敏感的,通过保持骨干网络参数冻结,避免了在3D任务中重新训练可能导致的特征坍塌

**优势**:

1. **表征保持**: 完整保留预训练学到的视觉知识
2. **稳定性**: 避免灾难性遗忘
3. **泛化性**: 保持模型在新任务上的迁移能力

#### 2.5.2 平面特定索引(RoPE Re-indexing)

对于使用旋转位置嵌入(RoPE)的模型(如DINOv3),PlaneCycle引入了巧妙的位置编码重索引机制。

**挑战**:
- RoPE编码基于2D位置
- 不同平面的切片具有不同的坐标系统

**解决方案**:
为当前激活的平面**重新计算Token坐标索引**

**实现**:

```python
def rope_reindex(tokens, plane, original_shape):
    """
    根据当前平面重新计算位置索引
    不引入可学习参数
    """
    D, H, W = original_shape

    if plane == 'HW':
        # 保持原始HW坐标
        positions = generate_2d_positions(H, W)
    elif plane == 'DW':
        # 将DW映射到2D坐标空间
        positions = generate_2d_positions(D, W)
    else:  # DH
        # 将DH映射到2D坐标空间
        positions = generate_2d_positions(D, H)

    return apply_rope_encoding(tokens, positions)
```

**效果**:
- 确保位置编码与当前处理的2D平面逻辑一致
- 完美兼容预训练模型对2D空间结构的理解偏好
- 不引入可学习参数

#### 2.5.3 全局Token处理

为了处理跨平面时的Token长度差异,引入了两种策略:

1. **全局均值(PCm - PlaneCycle Mean)**
   - 通过自适应平均池化
   - 维持层间Token统计分布的一致性
   - 适用于大多数场景

2. **分组策略(PCg - PlaneCycle Group)**
   - 将Token分组处理
   - 保持更多的局部信息
   - 适用于细粒度任务

**数学表达**:

$$
\text{PCm}(X) = \text{AdaptiveAvgPool}(X, \text{target\_size})
$$

$$
\text{PCg}(X) = \text{GroupNorm}(\text{Reshape}(X, \text{groups}))
$$

### 2.6 算法优势总结

PlaneCycle的核心优势:

1. **零成本迁移**: 无需训练,无需额外参数
2. **架构通用**: 适用于任何2D网络
3. **性能保证**: 保留预训练归纳偏置
4. **计算高效**: 复用2D计算,避免3D计算开销
5. **即插即用**: 可以无缝集成到现有系统中

3. 跨不同的任务(导航、操作、交互等)

实现统一的空间理解与推理能力。

### 3.3 空间聚合的实现

PlaneCycle通过创新的架构设计实现了高效的空间聚合。

#### 3.3.1 正交平面循环

**核心机制**: 将3D特征图循环分布在三个正交平面上

- **HW(轴向)**: 捕捉水平方向的空间关系
- **DW(冠状)**: 捕捉前后方向的空间关系
- **DH(矢状)**: 捕捉左右方向的空间关系

#### 3.3.2 层间渐进融合

**实现方式**:

1. **层级交替**: 在不同层级切换聚合平面
2. **信息累积**: 随着深度增加,融合所有平面的信息
3. **全局理解**: 最终形成完整的3D空间表征

**数学表达**:

$$
F^{(l)} = \text{Aggregate}(F^{(l-1)}, \text{Plane}^{(l)})
$$

其中, $F^{(l)}$ 是第l层的特征, $\text{Plane}^{(l)}$ 是第l层使用的平面。

#### 3.3.3 潜在3D世界帧

在动态模拟场景中,通过3D-VAE实现:

1. **环境编码**: 将环境编码为隐式体素网格(Latent Voxel Grid)
2. **时空注意力**: 在时空注意力机制下进行特征聚合
3. **持久表示**: 确保空间表示的持久性和一致性

## 4. 深入分析:为什么不需要Adapter?

基于前面的分析,我们深入探讨PlaneCycle能够实现无适配器迁移的内在机制。

### 4.1 架构设计视角

PlaneCycle能够实现**无须适配器(Adapter-free)**且**免训练(Training-free)**的2D到3D迁移,其核心逻辑在于它不改变模型参数,而是通过改变数据的"视图"来诱导预训练模型进行跨维度推理。

#### 4.1.1 循环正交平面

**核心思想**: 将3D体积数据视为一系列2D切片的集合,并在网络深度方向上交替改变这些切片的切分轴

**实现**:

1. **多视角观察**: 在不同层级,模型从不同平面观察3D数据
2. **信息互补**: 每个平面提供不同的空间信息
3. **融合效应**: 通过层级堆叠实现信息融合

#### 4.1.2 渐进式融合

**网络级提升**:

- **单层**: 只负责一个特定平面的2D聚合
- **多层**: 随着层数加深,信息在正交方向上不断交织
- **全局效果**: 单一的2D层能够产生**全局连贯的3D效果**

**数学表达**:

$$
F_{\text{3D}} = \prod_{l=1}^{L} \text{Aggregate}_{\text{Plane}^{(l)}}(F^{(l-1)})
$$

其中, $\prod$ 表示层的堆叠, $\text{Plane}^{(l)}$ 是第l层使用的平面。

### 4.2 特征对齐视角

训练自由的方法通过强制执行空间一致性来保证性能。

#### 4.2.1 打破"切片孤岛"

**传统方法的问题**:

- **逐切片推理**: 虽然保留了2D语义,但切片间相互隔离
- **特征不连贯**: 特征在深度方向上不连贯
- **3D信息丢失**: 无法捕捉完整的3D上下文

**PlaneCycle的解决**:

- **多平面循环**: 强制特征在多个平面上保持一致
- **内在融合**: 通过架构设计实现自动融合
- **结构化表征**: 产生具有3D结构化一致性的特征

#### 4.2.2 内在融合能力

**实验观察**:

PlaneCycle产生的特征在三个正交平面上都能表现出良好的对齐性

**原理分析**:

1. **一致性约束**: 模型在不同深度"看"到的平面方向不同
2. **共同规律**: 特征被迫在正交轴之间寻找共同的结构化规律
3. **结构化学习**: 即使不训练,也能产生判别力强的特征图

**数学证明**:

假设 $F_{HW}$, $F_{DW}$, $F_{DH}$ 分别是三个平面的特征,PlaneCycle通过最小化:

$$
\mathcal{L}_{\text{align}} = \| F_{HW} - F_{DW} \| + \| F_{DW} - F_{DH} \| + \| F_{DH} - F_{HW} \|
$$

实现特征对齐。由于这是架构固有的约束,不需要显式优化。

### 4.3 归纳偏置保留视角

PlaneCycle的高性能源于对2D基础模型在大规模数据中获得的**归纳偏置(Inductive Biases)**的完整保留。

#### 4.3.1 直接权重重用

**机制**:

- **无新参数**: 不引入新的参数
- **切片处理**: 将3D数据的每一个2D切片直接输入冻结的预训练层
- **知识迁移**: 利用2D模型对自然图像中空间分布、边缘和纹理的深刻理解

**效果**:

1. **完整保留**: 所有预训练的知识都被保留
2. **稳定推理**: 避免重新训练带来的不确定性
3. **泛化能力**: 保持在新任务上的迁移能力

#### 4.3.2 平面特定位置编码

**RoPE Re-indexing机制**:

对于使用旋转位置嵌入(RoPE)的模型:

1. **挑战**: RoPE编码基于2D位置,不同平面的切片具有不同的坐标系统
2. **解决**: 为当前激活的平面**重新计算Token的坐标索引**
3. **特点**: 不涉及学习,确保2D模型能够以正确的空间偏好处理3D数据

**实现细节**:

```python
def plane_specific_rope(tokens, plane, shape_3d):
    """
    平面特定的RoPE重索引
    """
    D, H, W = shape_3d

    # 根据平面生成2D坐标
    if plane == 'HW':
        coords = generate_grid(H, W)
    elif plane == 'DW':
        coords = generate_grid(D, W)
    else:  # DH
        coords = generate_grid(D, H)

    # 应用RoPE编码
    encoded = apply_rotary_embedding(tokens, coords)
    return encoded
```

**效果**:

- **空间一致性**: 确保位置编码与当前平面逻辑一致
- **偏好保持**: 完美兼容预训练模型对2D空间结构的理解
- **零成本**: 不引入任何可学习参数

#### 4.3.3 全局Token统计一致性

**问题**: 跨平面切换时,Token长度可能不同,导致特征分布漂移

**解决方案**:

1. **全局均值(PCm)**:
   - 通过自适应平均池化
   - 维持Token统计分布的一致性
   - 适用于大多数场景

2. **分组策略(PCg)**:
   - 将Token分组处理
   - 保持更多的局部信息
   - 适用于细粒度任务

**数学表达**:

$$
\text{PCm}(X) = \frac{1}{N} \sum_{i=1}^{N} X_i
$$

$$
\text{PCg}(X) = [\text{Mean}(X_{g_1}), \text{Mean}(X_{g_2}), ..., \text{Mean}(X_{g_k})]
$$

PlaneCycle证明了3D表征能力可以从预训练的2D基础模型中被**直接"解锁"**。

## 6. 技术实现细节

### 6.1 核心代码结构

```python
class PlaneCycle(nn.Module):
    def __init__(self, backbone_2d, num_layers):
        super().__init__()
        self.backbone = backbone_2d
        self.num_layers = num_layers
        self.planes = ['HW', 'DW', 'DH', 'HW']

    def forward(self, x_3d):
        """
        x_3d: [B, D, H, W, C]
        """
        B, D, H, W, C = x_3d.shape

        for i, layer in enumerate(self.backbone.layers):
            plane = self.planes[i % len(self.planes)]

            # 重塑为2D切片
            x_2d = self.reshape_to_2d(x_3d, plane)

            # RoPE重索引
            if hasattr(layer, 'rope'):
                x_2d = self.rope_reindex(x_2d, plane, (D, H, W))

            # 通过2D层处理
            x_2d = layer(x_2d)

            # 还原为3D
            x_3d = self.restore_to_3d(x_2d, plane, (D, H, W))

            # 全局Token处理
            x_3d = self.global_token_process(x_3d)

        return x_3d

    def reshape_to_2d(self, x_3d, plane):
        """根据平面重塑为2D切片"""
        B, D, H, W, C = x_3d.shape

        if plane == 'HW':
            # [B, D, H, W, C] -> [B*D, H, W, C]
            x_2d = x_3d.reshape(B * D, H, W, C)
        elif plane == 'DW':
            # [B, D, H, W, C] -> [B*H, D, W, C]
            x_2d = x_3d.permute(0, 2, 1, 3, 4).reshape(B * H, D, W, C)
        else:  # DH
            # [B, D, H, W, C] -> [B*W, D, H, C]
            x_2d = x_3d.permute(0, 3, 1, 2, 4).reshape(B * W, D, H, C)

        return x_2d

    def restore_to_3d(self, x_2d, plane, shape_3d):
        """从2D还原为3D"""
        B_total, _, _, C = x_2d.shape
        D, H, W = shape_3d
        B = B_total // (D if plane == 'HW' else (H if plane == 'DW' else W))

        if plane == 'HW':
            x_3d = x_2d.reshape(B, D, H, W, C)
        elif plane == 'DW':
            x_3d = x_2d.reshape(B, H, D, W, C).permute(0, 2, 1, 3, 4)
        else:  # DH
            x_3d = x_2d.reshape(B, W, D, H, C).permute(0, 2, 3, 1, 4)

        return x_3d

    def rope_reindex(self, x, plane, shape_3d):
        """平面特定的RoPE重索引"""
        D, H, W = shape_3d

        if plane == 'HW':
            coords = self.generate_2d_coords(H, W)
        elif plane == 'DW':
            coords = self.generate_2d_coords(D, W)
        else:  # DH
            coords = self.generate_2d_coords(D, H)

        return apply_rotary_pos_emb(x, coords)

    def global_token_process(self, x, method='mean'):
        """全局Token处理"""
        if method == 'mean':
            return x  # 默认不处理,保持Token统计
        elif method == 'group':
            # 分组平均
            return self.group_average(x)
```

class PlaneCycleCNN(PlaneCycle):

        super().__init__(cnn_model, len(cnn_model.stages))

    def forward(self, x_3d):
        # 初始卷积
        x = self.initial_conv_3d(x_3d)

        # PlaneCycle处理
        for i, stage in enumerate(self.backbone.stages):

        return x

## 8. 局限性与未来工作

### 8.1 当前局限

#### 8.1.1 数据特性依赖

**问题**: 性能受数据的长宽比影响

**表现**:
- 当D、H、W接近时,性能最优
- 当某个维度远大于其他时,性能下降

**原因**: 循环平面策略假设各维度同等重要

#### 8.1.2 稀疏数据效率

**问题**: 对稀疏3D数据效率低

**表现**:
- 大部分体素为空
- 计算资源浪费在空区域

**原因**: 切片方法处理所有体素,包括空体素

#### 8.1.3 时序建模缺失

**问题**: 无法直接处理动态3D数据

**表现**:
- 需要额外的时序建模模块
- 对于视频3D数据效果有限

**原因**: PlaneCycle专注于空间维度,未考虑时间维度

### 8.2 改进方向

#### 8.2.1 自适应平面选择

**思路**: 根据数据特性动态选择平面

class AdaptivePlaneCycle(PlaneCycle):
