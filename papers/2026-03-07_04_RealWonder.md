## 核心信息

### 摘要

当前的视频生成模型无法模拟3D动作（如力和机器人操作）的物理后果，因为它们缺乏对动作如何影响3D场景的结构化理解。RealWonder提出了首个从单张图像进行动作条件视频生成的实时系统。

**核心洞察**：使用物理仿真作为中间桥梁——不是直接编码连续动作，而是通过物理仿真将它们转换为视频模型可以处理的视觉表示（光流和RGB）。

RealWonder集成了三个组件：
1. 从单张图像进行3D场景重建
2. 物理仿真
3. 仅需4步扩散的蒸馏视频生成器

**性能**：在480×832分辨率下达到13.2 FPS，能够对刚体、可变形体、流体和颗粒材料进行力、机器人动作和相机控制的交互式探索。

**应用场景**：运动规划、AR/VR、机器人学习

### 关键贡献

1. **首个实时流式系统**：提出RealWonder，第一个接受3D物理动作（包括力、扭矩、机器人动作和相机控制）作为输入的实时动作条件视频生成系统，在480×832分辨率下达到13.2 FPS。

2. **物理仿真桥梁架构**：引入了一种新颖的架构，使用物理仿真作为中间表示，能够在不训练动作-视频对或不标记动作的情况下预测3D物理动作后果。

3. **流条件蒸馏方案**：设计了一个蒸馏方案，将光流条件整合到视频生成中，显著减少扩散开销同时实现有效的流控制。

## 核心技术发现

### 1. 物理仿真作为中间表示桥梁

**核心问题**：
- 视频扩散模型擅长理解像素或潜在空间中的视觉模式，但缺乏解释3D力如何在3D场景中传播的结构化理解
- 物理动作（如力和扭矩）是连续且无界的，难以用适用于离散输入（如相机姿态）的标记化方案处理
- 获取用于训练的动作-视频对仍然是一个开放问题

**解决方案**：
使用物理仿真器将动作后果转换为视频模型可以自然处理的视觉运动模式：
1. 给定3D物理动作，首先通过物理仿真预测其后果
2. 将结果渲染为光流和粗糙的RGB预览
3. 这些中间表示保留了动作和结果之间的因果关系，同时提供现代视频生成器设计处理的视觉信号

**关键优势**：
- 优雅地避免了标记化问题，通过物理仿真器将连续动作信号转换为离散像素
- 不需要动作-视频对，只需要2D流-视频对来训练视频生成器

### 2. 三阶段流水线架构

#### Stage 1: 单图像3D场景重建

**场景表示**：S = B ∪ O
- 静态背景 B
- 动态对象 O

**背景重建**：
- 表示为点云 B = {(p_i^B, c_i^B)}_{i=1}^{N_B}
- 通过分割静态区域、修复被对象遮挡的区域、估计每像素深度、反投影到3D空间
- 这些点在仿真期间作为静态碰撞边界

**对象重建**：
- 表示为点云 O = {(p_j^O, c_j^O, v_j)}_{j=1}^{N_O}
- 包括刚体、布料、颗粒材料和流体
- 使用前馈重建模型生成完整的3D网格
- 通过姿态估计和比例对齐将网格注册到场景坐标系
- 提取不可见表面的网格顶点以补充反投影的对象像素

**材料估计**：
- 使用视觉-语言模型（VLM）将每个对象分类为六种材料类别之一（刚体、弹性、布料、烟雾、液体、颗粒）
- 估计相应的物理参数（密度、摩擦系数、弹性模量、粘度）

#### Stage 2: 物理仿真

**动作表示**：统一三种类型的3D场景空间动作
1. **外力** f_t(x,y,z) ∈ R^3：直接在指定的3D位置施加
2. **机器人末端执行器命令** r_t = {p_t^{ee}, q_t^{ee}, g_t}：指定位置、方向和夹持器状态，通过求解逆运动学（IK）转换为关节扭矩和力
3. **相机姿态** C_t = {R_t, t_t}：在渲染期间应用

**物理求解器**：
在每个时间步t，物理引擎获取当前场景状态S_t和动作a_t作为输入，计算所有动态点的更新位置p_{t+1}和速度v_{t+1}：

(p_{t+1}, v_{t+1}) = PhysicsStep(S_t, a_t)

**专用求解器**：
- **刚体动力学**：通过形状匹配进行碰撞处理
- **位置基动力学（PBD）**：用于弹性体、布料和烟雾
- **物质点法（MPM）**：用于液体和颗粒材料
- 这些求解器耦合以处理多材料交互

**性能**：单个物理步骤通常在不到2ms内完成

**中间表示**：

1. **光流** F_t ∈ R^{H×W×2}：
   - 通过投影3D速度场计算像素空间流
   - F_t(u,v) = Π(p_t + Δt·v_t) - Π(p_t)
   - 捕获应用动作的运动后果

2. **粗糙RGB渲染** Ṽ_t ∈ R^{H×W×3}：
   - 使用简单的点云光栅化渲染预览视频
   - 虽然视觉上近似，但提供了纯流无法捕获的关键结构线索（如遮挡变化）

**三个关键目标**：
1. 保留动作与其视觉后果之间的因果关系
2. 存在于视频模型可以处理的视觉域中
3. 可以实时计算

#### Stage 3: 实时条件视频生成

**输入**：输入图像I、文本提示text、光流F_t、粗糙预览Ṽ_t

**输出**：物理合理的实时视频 V_t = G(text, I, F_t, Ṽ_t)

**两阶段方法**：

**1. 流条件教师模型**：
- 从预训练的图像到视频扩散模型开始
- 通过后训练增强流控制
- 使用基于流的噪声扭曲：
  - 采样单帧高斯噪声 z ~ N(0,I)
  - 根据流场在时间上扭曲以获得结构化噪声 z^F = Warp(z, F)
  - 这种扭曲保留了高斯分布，同时将运动模式直接编码到噪声结构中
- 使用流匹配目标微调，以建模流扭曲噪声和数据分布之间的速度场
- 优势：通过初始噪声直接注入控制，保持效率同时实现精确的运动遵守

**2. 因果蒸馏用于流式传输**：
- 流条件教师仍然是需要完整序列处理的双向模型，与实时流式生成不兼容
- 将其蒸馏为仅需4个去噪步骤顺序生成帧的因果学生
- 遵循分布匹配蒸馏（DMD），最小化学生输出分布和教师之间的反向KL散度：
  ∇L_DMD = E_t[∇_θ KL(p_fake,t || p_real,t)]
- 采用自强迫训练范式，在训练期间进行自回归展开
- 通过在应用RoPE之前存储KV缓存并添加注意力汇来修复质量退化

### 3. 流式推理架构

**RGB条件通过SDEdit**：
- 在4步去噪过程中合并粗糙RGB预览Ṽ_t
- 不是从纯流扭曲噪声V_{t,(4)} = z_t^F一直去噪到干净样本x_0
- 而是从混合开始：
  V_{t,(3)} = α_{(3)}·E(Ṽ_t) + √(1-α_{(3)}^2)·z_t^F
- 通过从步骤3而不是步骤4开始去噪，允许模型一个初始步骤去噪混合，然后执行剩余的三个标准去噪步骤
- 这种双重条件保留了来自流的运动准确性，同时合并了来自物理预览的结构线索

**流式架构**：
- 维护两个并行流：
  1. 物理仿真及其渲染为中间表示
  2. 视频生成以13.2 FPS运行，消耗最新的物理衍生条件
- 蒸馏模型G的因果特性启用了逐帧生成：
  V_{t+1} = G(text, I, F_{t+1}, Ṽ_{t+1}, {V_j}_{j≤t})

## 📝 核心问题分析

### Q1: 物理仿真如何桥接3D重建和视频生成？

**详细分析**：

#### 1) 3D重建提供了什么输入

3D重建模块为物理仿真提供了完整的3D场景表示：

**静态背景**：
- 点云表示：B = {(p_i^B, c_i^B)}_{i=1}^{N_B}
- 3D位置 p_i^B ∈ R^3 和RGB颜色 c_i^B ∈ R^3
- 通过以下步骤获得：
  - 分割静态区域
  - 修复被对象遮挡的区域
  - 估计每像素深度
  - 反投影到3D空间
- 作为仿真期间的静态碰撞边界

**动态对象**：
- 点云表示：O = {(p_j^O, c_j^O, v_j)}_{j=1}^{N_O}
- 包括3D位置、RGB颜色和速度v_j ∈ R^3
- 对象类型包括：刚体、布料、颗粒材料、流体
- 通过以下步骤获得：
  - 使用前馈重建模型生成完整3D网格
  - 通过姿态估计和比例对齐注册到场景坐标系
  - 提取不可见表面的网格顶点（如对象背面）
  - 补充反投影的对象像素

**材料属性**：
- 使用视觉-语言模型（VLM）分类为六种材料类别
- 估计物理参数：
  - 密度
  - 摩擦系数
  - 弹性模量
  - 粘度
- 用户可以覆盖这些分类和参数以进行创意控制或改进物理准确性

#### 2) 物理仿真如何处理这些输入

**统一动作表示**：

物理仿真器接受三种类型的3D场景空间动作：

1. **外力** f_t(x,y,z) ∈ R^3：
   - 直接在指定的3D位置施加
   - 可以是点力或力场（如风）

2. **机器人末端执行器命令** r_t = {p_t^{ee}, q_t^{ee}, g_t}：
   - p_t^{ee}：末端执行器位置
   - q_t^{ee}：末端执行器方向
   - g_t：夹持器状态
   - 通过逆运动学（IK）转换为关节扭矩和力
   - 驱动仿真中的关节机器人模型

3. **相机姿态** C_t = {R_t, t_t}：
   - R_t：旋转矩阵
   - t_t：平移向量
   - 在渲染期间应用

**物理求解步骤**：

在每个时间步t，物理引擎执行：

(p_{t+1}, v_{t+1}) = PhysicsStep(S_t, a_t)

其中：
- S_t：当前场景状态（背景+对象+材料属性）
- a_t：当前动作（力/机器人/相机）
- p_{t+1}：更新的3D位置
- v_{t+1}：更新的速度

**专用求解器**：

根据材料类型使用不同的物理求解器：

1. **刚体动力学**：
   - 使用形状匹配进行碰撞处理
   - 适用于：刚性对象（如柿子、船）

2. **位置基动力学（PBD）**：
   - 用于弹性体、布料和烟雾
   - 适用于：可变形对象（如布料、弹性物体）

3. **物质点法（MPM）**：
   - 用于液体和颗粒材料
   - 适用于：流体（如水）、颗粒（如沙子、雪）

4. **多材料耦合**：
   - 这些求解器耦合以处理多材料交互
   - 例如：机器人夹持器与布料交互、风吹动沙堡

**性能**：
- 单个物理步骤通常在不到2ms内完成
- 仿真和渲染流以30 FPS运行，与视频生成器流并行

**时间步长**：
- 仿真时间步长：0.01s
- 每个仿真步骤最多20个子步骤以确保数值稳定性

#### 3) 仿真结果如何转换为视频帧

**中间表示生成**：

物理仿真输出被转换为两种视觉表示，作为视频生成的条件信号：

**1. 光流（Optical Flow）**：

计算公式：
F_t(u,v) = Π(p_t + Δt·v_t) - Π(p_t)

其中：
- F_t ∈ R^{H×W×2}：像素空间流
- Π：相机投影函数
- (u,v)：像素坐标
- p_t：当前3D位置
- v_t：当前速度
- Δt：时间步长

**含义**：
- 捕获应用动作的运动后果
- 表示每个像素在下一帧中应该移动到哪里
- 编码了动作和结果之间的因果关系

**2. 粗糙RGB渲染（Coarse RGB Rendering）**：

渲染过程：
- 使用简单的点云光栅化
- 渲染预览视频 Ṽ_t ∈ R^{H×W×3}
- 虽然视觉上近似，但提供了关键的结构线索

**重要性**：
- 提供纯流无法捕获的线索：
  - 遮挡变化
  - 对象变形
  - 表面交互

**视频生成过程**：

**输入到视频生成器**：
- 输入图像I
- 文本提示text
- 光流F_t（来自物理仿真）
- 粗糙RGB预览Ṽ_t（来自物理仿真）

**生成公式**：
V_t = G(text, I, F_t, Ṽ_t)

**两阶段生成**：

**阶段1：流条件教师模型**
- 从预训练的图像到视频扩散模型开始（VideoXFun wan2.1-1.3B-InP）
- 通过LoRA后训练增强流控制（rank=2048）
- 训练300K迭代，学习率10^-5
- 使用基于流的噪声扭曲：
  - 采样单帧高斯噪声 z ~ N(0,I)
  - 根据流场在时间上扭曲：z^F = Warp(z, F)
  - 流匹配目标微调

**阶段2：因果蒸馏**
- 将多步教师蒸馏为4步因果学生
- 启用流式生成
- 分布匹配蒸馏（DMD）：
  ∇L_DMD = E_t[∇_θ KL(p_fake,t || p_real,t)]
- 自强迫训练范式
- 存储RoPE之前的KV缓存
- 添加注意力汇以稳定长序列生成

**SDEdit合并RGB预览**：

不是从纯流扭曲噪声开始，而是从混合开始：

V_{t,(3)} = α_{(3)}·E(Ṽ_t) + √(1-α_{(3)}^2)·z_t^F

其中：
- E：VAE编码器
- α_{(3)}：扩散步骤3的噪声调度系数
- z_t^F：流扭曲噪声

**优势**：
- 保留来自流的运动准确性
- 合并来自物理预览的结构线索
- 导致一致的对象变形和遮挡处理

**最终输出**：
- 逐帧生成：V_{t+1} = G(text, I, F_{t+1}, Ṽ_{t+1}, {V_j}_{j≤t})
- 13.2 FPS的实时流式生成
- 480×832分辨率

#### 4) 这个桥接过程的关键技术是什么

**技术1：物理仿真作为中间表示**

**为什么需要**：
- 视频扩散模型擅长视觉模式，但缺乏3D结构理解
- 物理动作是连续且无界的，难以标记化
- 动作-视频对难以获取

**解决方案**：
- 物理仿真器自然处理无界的力和动作输入
- 将连续动作信号转换为离散像素
- 保留动作和结果之间的因果关系

**技术2：光流作为条件信号**

**优势**：
- 光流存在于视觉域中，视频模型可以处理
- 编码运动模式而不需要显式的3D理解
- 可以通过基于流的噪声扭曲直接注入到扩散过程中

**实现**：
- 从3D速度场投影到2D像素空间
- 流扭曲噪声：z^F = Warp(z, F)
- 流匹配目标训练

**技术3：双条件信号（流+RGB）**

**为什么需要两者**：
- 光流：提供运动准确性
- RGB预览：提供结构线索（遮挡、变形）

**SDEdit合并**：
- 从步骤3而不是步骤4开始去噪
- 混合编码的RGB预览和流扭曲噪声
- 一个初始步骤去噪混合，然后三个标准去噪步骤

**技术4：因果蒸馏**

**挑战**：
- 标准扩散模型需要完整序列处理
- 不兼容实时流式生成

**解决方案**：
- 将多步教师蒸馏为4步因果学生
- 分布匹配蒸馏（DMD）
- 自强迫训练范式
- KV缓存 + 注意力汇

**结果**：
- 从50步减少到4步
- 启用逐帧生成
- 保持生成质量

**技术5：多材料物理求解器耦合**

**支持的材料**：
- 刚体：形状匹配
- 弹性体/布料/烟雾：位置基动力学（PBD）
- 液体/颗粒：物质点法（MPM）

**耦合**：
- 处理多材料交互
- 例如：机器人夹持器与布料、风吹沙堡

**技术6：实时流水线架构**

**双并行流**：
1. 物理仿真流（30 FPS）：
   - 仿真场景动态
   - 渲染光流和RGB预览

2. 视频生成流（13.2 FPS）：
   - 消耗最新物理衍生条件
   - 生成照片级真实帧

**性能**：
- 物理步骤：<2ms
- 总体：13.2 FPS
- 延迟：<100ms每帧

**技术7：避免动作-视频对训练**

**传统方法的问题**：
- 动作-视频对稀缺
- 从观察到的视频中推断精确的物理动作通常不可行

**RealWonder的解决方案**：
- 只需要2D流-视频对训练视频生成器
- 使用物理仿真器生成流
- 不需要真实的动作-视频对

**训练数据**：
- 200K对流-扭曲噪声和视频对
- 180K真实世界剪辑（OpenVid，80-120帧）
- 20K合成视频（Wan2.1-14B-T2V，VidProM提示）

**总结**：

物理仿真桥接3D重建和视频生成的关键是通过**中间表示（光流+RGB）**将**物理动作的后果**转换为**视频模型可以理解的视觉信号**，同时：

1. 保留因果关系（动作→结果）
2. 避免标记化问题（连续→离散）
3. 避免动作-视频对（流-视频对即可）
4. 实现实时性能（2ms物理+4步扩散）
5. 支持多材料交互（刚体/弹性/流体/颗粒）
6. 双条件信号（流+RGB）

### Q3: 如何实现13.2 FPS的实时性能？

**详细技术分析**：

#### 1. 系统性能分解

**总体性能**：
- **帧率**：13.2 FPS
- **分辨率**：480×832
- **延迟**：<100ms每帧
- **硬件**：单个H200 GPU

**各模块性能**：

**模块1：3D重建（一次性）**
- 深度估计：~100ms（使用现成模型）
- 点云生成：~50ms
- 材料分类：~20ms（VLM）
- **总计**：~170ms（仅在开始时执行一次）

**模块2：物理仿真（持续）**
- 单步仿真：<2ms
- 流式仿真：30 FPS
- 多材料耦合：<5ms
- **总计**：~33ms每步（30 FPS）

**模块3：中间表示生成（持续）**
- 光流计算：~5ms（从3D速度投影）
- RGB渲染：~10ms（点云光栅化）
- **总计**：~15ms

**模块4：视频生成（持续）**
- 4步扩散：~60ms（主要瓶颈）
- VAE编码/解码：~10ms
- **总计**：~70ms

**流水线总时间**：
- 物理仿真（33ms）|| 视频生成（70ms）
- **瓶颈**：视频生成（70ms）
- **实际帧率**：1000ms / 70ms ≈ 14.3 FPS
- **报告帧率**：13.2 FPS（考虑其他开销）

#### 2. 关键优化技术

**优化1：因果蒸馏（从50步到4步）**

**问题**：
- 标准扩散模型需要50步去噪
- 每步~15ms，总计~750ms
- 远低于实时要求（<100ms）

**解决方案**：分布匹配蒸馏（DMD）

**训练过程**：

**阶段1：流条件教师模型**
- 基础模型：VideoXFun wan2.1-1.3B-InP
- LoRA微调（rank=2048）
- 训练：300K迭代，学习率10^-5
- 输出：50步教师模型

**阶段2：因果适应**
- 从教师采样2K ODE轨迹
- 训练学生模型：3K迭代，MSE损失
- 目标：学习单步生成

**阶段3：分布匹配**
- 训练：600迭代
- 损失：KL散度 min KL(p_student || p_teacher)
- 目标：匹配分布

**自强迫训练**：
- 在训练期间进行自回归展开
- 模拟推理时的行为
- 避免训练-测试差距

**稳定性改进**：
- 存储RoPE之前的KV缓存
- 添加注意力汇
- 防止长序列质量退化

**结果**：
- 从50步减少到4步
- 速度提升：12.5×
- 质量保持：接近教师模型

**优化2：并行流水线架构**

**设计**：
- 两个并行流：
  1. 物理仿真流（30 FPS）
  2. 视频生成流（13.2 FPS）

**优势**：
- 物理仿真和视频生成并行执行
- 视频生成器消耗最新的物理条件
- 避免串行等待

**实现**：

**流1：物理仿真**（33ms周期）
```
while streaming:
    action = get_user_action()
    scene_state = physics_step(scene_state, action)
    flow = compute_optical_flow(scene_state)
    rgb_preview = render_point_cloud(scene_state)
    put_conditions(flow, rgb_preview)
```

**流2：视频生成**（70ms周期）
```
while streaming:
    flow, rgb_preview = get_latest_conditions()
    frame = video_generator(flow, rgb_preview, history)
    output_frame(frame)
```

**同步**：
- 视频生成器总是使用最新的物理条件
- 如果物理仿真更快，丢弃中间结果
- 如果视频生成更快，等待新条件

**优化3：轻量级物理求解器**

**Genesis仿真器**：
- 通用物理引擎
- 支持多材料（刚体、弹性、流体、颗粒）
- 优化的CUDA实现

**求解器选择**：

**刚体**：
- 形状匹配碰撞检测
- 时间复杂度：O(n log n)
- 典型时间：<1ms（1000个对象）

**弹性体/布料**（PBD）：
- 位置基动力学
- 迭代求解器（10-20次迭代）
- 典型时间：<2ms（10000个粒子）

**流体/颗粒**（MPM）：
- 物质点法
- 网格-粒子混合表示
- 典型时间：<5ms（50000个粒子）

**优化技术**：
- 空间哈希用于碰撞检测
- 并行粒子更新
- 自适应子步（最多20步）

**时间步长**：
- 主步长：0.01s
- 子步：最多20步（确保稳定性）
- 实际步长：0.0005s（数值稳定）

**优化4：高效的光流计算**

**方法**：3D速度场投影

**公式**：
F_t(u,v) = Π(p_t + Δt·v_t) - Π(p_t)

**步骤**：
1. 获取当前3D位置p_t和速度v_t（来自物理仿真）
2. 计算下一帧的3D位置：p_t + Δt·v_t
3. 投影到2D：Π(p_t)和Π(p_t + Δt·v_t)
4. 计算差值：光流F_t

**优势**：
- 直接从3D物理获得，无需光流估计
- 时间复杂度：O(n)，n为点数
- 典型时间：~5ms（100000个点）

**实现**：
- 并行投影（CUDA）
- Z-buffer处理遮挡
- 双线性插值填充空洞

**优化5：点云光栅化**

**方法**：简单的点云渲染

**步骤**：
1. 将点云投影到2D
2. 使用点的颜色
3. 处理遮挡（Z-buffer）
4. 可选： splatting提高质量

**优势**：
- 非常快速（~10ms）
- 不需要完整的网格渲染
- 提供足够的结构线索

**质量权衡**：
- 视觉上粗糙
- 但足够提供遮挡和变形线索
- 高质量由视频生成器完成

**优化6：流扭曲噪声**

**方法**：基于流的噪声扭曲

**步骤**：
1. 采样单帧高斯噪声：z ~ N(0,I)
2. 根据流场扭曲：z^F = Warp(z, F)
3. 使用扭曲噪声作为扩散起点

**优势**：
- 直接注入运动信息
- 无需额外的条件模块
- 保持高斯分布（数学性质）

**效率**：
- 扭曲操作：~5ms
- 双线性插值
- 并行处理（CUDA）

**优化7：SDEdit合并RGB**

**方法**：从步骤3开始去噪

**公式**：
V_{t,(3)} = α_{(3)}·E(Ṽ_t) + √(1-α_{(3)}^2)·z_t^F

**步骤**：
1. 编码RGB预览：E(Ṽ_t)
2. 混合编码和流扭曲噪声
3. 从步骤3开始去噪（而不是步骤4）
4. 执行4步去噪（实际是3步+1步混合）

**效率**：
- VAE编码：~5ms
- 混合：~1ms
- 不增加去噪步骤

**质量提升**：
- 更好的遮挡处理
- 更一致的对象变形
- 更高的视觉质量

#### 3. 内存和计算优化

**模型大小**：
- 基础模型：1.3B参数
- LoRA rank：2048
- 总参数：~1.35B

**内存占用**：

**批处理**：
