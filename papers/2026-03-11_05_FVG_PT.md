## 论文概述

### 摘要

FVG-PT（Foreground View-Guided Prompt Tuning）是一种自适应的即插即用型前景注意力引导模块，旨在解决视觉语言模型（VLMs）中prompt tuning过程导致的视觉编码器前景注意力偏移问题。

**核心贡献**：
1. **前景注意力偏移分析**：首次系统性地将prompt tuning的失败模式归因于视觉编码器前景注意力的偏移
2. **前景可靠性门控（FRG）**：自适应评估前景视图质量的机制
3. **前景蒸馏补偿（FDC）**：引导视觉注意力聚焦于前景的模块
4. **先验校准（PC）**：解决base-new trade-off问题的策略

### 主要创新点

1. **即插即用架构**：可以应用于多种prompt tuning方法（CoOp, KgCoOp, PromptSRC, MMRL等）
2. **自适应机制**：通过学习型门控机制动态调整对前景视图和CLIP先验的信任度
3. **参数高效**：仅需0.13M可训练参数，优于现有backbone方法
4. **全解耦设计**：在logit层面完全解耦base和new分支，有效缓解BNT问题

## 研究背景与动机

### Prompt Tuning的兴起

视觉语言模型（VLMs）如CLIP在跨模态对齐和零样本泛化方面表现出色。然而，全量微调这些大型模型成本高昂。**Prompt Tuning**作为参数高效的替代方案，通过引入轻量级可学习的prompt向量作为输入查询，引导模型输出适应目标任务分布。

现有研究主要关注：
- **文本prompts**：CoOp, CoCoOp
- **视觉prompts**：VPT, SA2VP
- **跨模态prompts**：PromptSRC, DPC
- **中间层插件**：MMA, MMRL

### 核心问题：注意力偏移

**关键发现**：现有方法在优化可学习prompts时，**忽略了prompt tuning过程对VLM内部注意力表示的影响**。

#### 理论分析

在CLIP的对比学习框架中：
- **文本prompt作为查询**：决定了图像patch tokens在特征空间中的响应权重
- **输入prompts直接影响**：图像内部的注意力分布

**推论**：随着可学习prompts的持续优化，相应的图像注意力分布也会发生变化。

#### 实验验证（Grad-CAM分析）

论文通过Grad-CAM可视化分析发现：
- **错误预测样本**：视觉ViT编码器的注意力常常从主要对象（前景）偏移到无关的背景
- **前景语义对齐失败**：导致与文本的语义对齐失败
- **定量分析**：性能更高的模型倾向于展现更小的前景偏移

**结论**：prompt tuning的许多错误预测可以归因于视觉编码器前景注意力分配的偏移。

### 现有方法的局限性

虽然已有研究尝试引导VLM注意力聚焦于前景，但存在以下关键局限：

1. **前景视图质量不可控**：
   - 可能不正确、不完整或过大
   - 导致模型学习不准确的alignment模式

2. **Base-New Trade-off (BNT)问题**：
   - 过度关注前景可能削弱背景中的常识特征
   - 损害对新类别的泛化能力

3. **缺乏自适应机制**：
   - 无法灵活调整对前景视图的信任度
   - 无法动态平衡base-class适应和new-class泛化

## 核心问题1：算法原理

### Q1：核心算法流程是什么？

FVG-PT的核心算法包含三个主要模块，形成完整的自适应前景引导prompt tuning框架。

### 1. Foreground Reliability Gate (FRG) - 前景可靠性门控

#### 目标
自适应评估前景视图的质量，学习动态权重方案。

#### 实现机制

**Step 1：前景提取**
```python
# 使用预训练的SEEM分割模型
m = SEEM(x)  # 前景mask ∈ {0,1}^{H×W}
x_fg = x ⊙ m  # 前景视图 = 原始图像 ⊙ mask
```

**Step 2：监督信号构建**
```python
# 使用backbone模型获得硬监督
z_full = backbone(x)  # 完整图像的预测logits
z_fg = backbone(x_fg)  # 前景视图的预测logits

l_full = CE(z_full, t)  # 完整图像的交叉熵损失
l_fg = CE(z_fg, t)      # 前景视图的交叉熵损失

# 二值监督目标
r* = I[l_fg < l_full]  # 如果前景损失更低，则r*=1
```

**Step 3：可靠性指标构建**

FRG使用3个可靠性指标（N_frg=3）：

1. **分布指标** - ΔH：
   ```
   ΔH = H(p_full) - H(p_fg)
   ```
   - 如果前景更可靠，p_fg更sharp，熵更低，ΔH > 0

2. **相似性指标** - cos(p_full, p_fg)：
   - 约束p_fg保持接近p_full
   - 降低由不准确masks导致的分布偏移风险

3. **几何指标** - A(m)：
   - 前景与完整图像的面积比
   - 确保前景足够信息量，不会过小

**Step 4：门控机制**
```python
# 输入向量
u = [ΔH, cos(p_full, p_fg), A(m)]

# MLP预测信任分数
q = MLP(u)  # 标量logit
r = sigmoid(q)  # 信任分数 ∈ (0,1)

# 训练损失（二值交叉熵）
L_FRG = -r* log(r) - (1-r*) log(1-r)
```

**关键特性**：
- **类别无关**：FRG与类别无关，可以评估任何前景视图输入的可靠性
- **自适应权重**：学习在多个准则下的组合权重方案
- **基于backbone**：使用backbone最终logits构建特征，确保即插即用兼容性

### 2. Foreground Distillation Compensation (FDC) - 前景蒸馏补偿

#### 目标
引导VLM注意力聚焦于过滤后的前景视图，纠正偏移的注意力。

#### 设计理念

**关键挑战**：前景引导的alignment模式与原始backbone不同，可能产生优化方向冲突。

**解决方案**：在backbone原始特征alignment阶段后插入轻量级adapter，学习前景导向的特征重投影模式，同时保持与backbone原始优化方向的兼容性。

#### 实现机制

**Step 1：特征重投影**

对backbone输出的图像特征f(x)和文本特征g(t)，引入带残差连接的两层瓶颈MLP作为adapter：

```python
# 视觉分支adapter
f_tilde(x) = L2Norm(f(x) + MLP_v(f(x)))

# 文本分支adapter
g_tilde(t) = L2Norm(g(t) + MLP_t(g(t)))
```

**设计要点**：
- **瓶颈结构**：隐藏层维度小于输入，减少参数量
- **双分支设计**：视觉和文本分支使用独立参数，避免单边微调破坏跨模态alignment
- **残差连接**：保留原始特征信息

**Step 2：FDC logit计算**
```python
z_FDC(x) = [f_tilde(x)]^T g_tilde(t)  # 补偿特征的图像-文本交互
p_FDC = softmax(z_FDC(x) / τ_d)       # 预测分布
```

**Step 3：蒸馏目标**

FDC使用KL散度蒸馏，目标分布根据信任分数r自适应选择：

```python
# 自适应蒸馏目标
L_dist = r · KL(p_fg || p_FDC) + (1-r) · KL(p_full || p_FDC)
```

- **r → 1**（前景可靠）：FDC更接近前景分布p_fg
- **r → 0**（前景不可靠）：FDC回退到完整图像分布p_full

**Step 4：总体损失**
```python
L_base = CE(z_FDC(x), t) + L_FRG + λ_d · L_dist
```

其中：
- CE：交叉熵损失，用于base-class判别
- L_FRG：FRG训练损失
- L_dist：蒸馏损失，λ_d = 10.0

### 3. Prior Calibration (PC) - 先验校准

#### 目标
解决BNT（Base-New Trade-off）问题，保持对new classes的泛化能力。

#### 问题分析

**BNT成因**：
- FDC的前景强调可能导致模型过度关注前景
- 背景中包含常识特征，被过度忽略会损害泛化能力
- 在base classes上过度拟合，降低new classes性能

#### 设计理念

**核心思想**：在logit层面完全解耦base和new分支，隔离优化路径。

**关键机制**：
1. **分支解耦**：new分支不调用FDC，完全隔离base分支的影响
2. **自适应CLIP先验平衡**：通过Backbone Reliability Gate学习backbone和CLIP先验的自适应权重

#### 实现机制

**Step 1：分支解耦**

推理阶段：
- **Base分支**：使用微调的FDC adapters计算重投影的图像-文本logits z_FDC(x)
- **New分支**：应用原始零样本CLIP，使用初始文本特征t_CLIP获得先验logits

```python
# Base分支（使用FDC）
z_FDC(x) = [f_tilde(x)]^T g_tilde(t)

# New分支（使用CLIP先验）
z_CLIP(x) = [f_CLIP(x)]^T g_CLIP(t_CLIP)
```

**Step 2：Backbone Reliability Gate (BRG)**

BRG学习样本自适应的CLIP信任分数b ∈ [0,1]：

```python
# 输入统计向量（N_brg=3）
s = [H(p_full), H(p_CLIP), cos(p_full, p_CLIP)]

# MLP预测
b = sigmoid(MLP(s))

# 校准logits
z_PC(x) = (1-b) · z_full(x) + b · z_CLIP(x)
```

**统计指标解读**：
- **H(p_CLIP)**：CLIP分布熵，低熵表示CLIP更自信，应增加b
- **H(p_full)**：Backbone分布熵，高熵表示backbone不确定，应增加b
- **cos(p_full, p_CLIP)**：分布相似度，相似度低时驱动b向更自信的一方

**Step 3：训练目标**
```python
L_PC = CE(z_PC(x), t) + KL(p_CLIP || softmax(z_PC(x)/τ_d))
```

- **CE损失**：base-class判别
- **KL正则**：保持校准分布接近CLIP先验

**训练行为**：
- BRG学习如何基于图像-文本特征分布分配权重
- 例如：H(p_CLIP)低或H(p_full)高 → CLIP更自信 → 分配更大的b
- 鼓励z_PC(x)遵循更可靠的分布

### 完整训练流程

```
输入: 图像-文本对 (x, t), 前景视图 x_fg

1. 前景提取:
   m = SEEM(x)
   x_fg = x ⊙ m

2. Base分支 (FRG + FDC):
   a) 计算可靠性指标 u = [ΔH, cos(p_full, p_fg), A(m)]
   b) FRG预测 r = sigmoid(MLP_FRG(u))
   c) 计算L_FRG = BCE(r*, r)
   
   d) Adapter特征重投影:
      f_tilde(x) = L2Norm(f(x) + MLP_v(f(x)))
      g_tilde(t) = L2Norm(g(t) + MLP_t(g(t)))
   
   e) 计算FDC logit和蒸馏损失:
      z_FDC = [f_tilde]^T g_tilde
      L_dist = r·KL(p_fg||p_FDC) + (1-r)·KL(p_full||p_FDC)
   
   f) Base总体损失:
      L_base = CE(z_FDC, t) + L_FRG + λ_d·L_dist

3. New分支 (PC):
   a) 计算CLIP先验:
      z_CLIP = [f_CLIP]^T g_CLIP(t_CLIP)
   
   b) BRG预测:
      s = [H(p_full), H(p_CLIP), cos(p_full, p_CLIP)]
      b = sigmoid(MLP_BRG(s))
   
   c) 校准logits:
      z_PC = (1-b)·z_full + b·z_CLIP
   
   d) New分支损失:
      L_PC = CE(z_PC, t) + KL(p_CLIP || softmax(z_PC/τ_d))

4. 联合优化:
   L_total = L_base + L_PC
```

### 推理流程

```
输入: 图像 x

1. Base分支:
   f_tilde(x) = L2Norm(f(x) + MLP_v(f(x)))
   g_tilde(t) = L2Norm(g(t) + MLP_t(g(t)))
   z_base = [f_tilde]^T g_tilde
   y_base = argmax(z_base)

2. New分支:
   z_CLIP = [f_CLIP]^T g_CLIP(t_CLIP)
   b = sigmoid(MLP_BRG(s))
   z_new = (1-b)·z_full + b·z_CLIP
   y_new = argmax(z_new)

输出: y_base (base classes), y_new (new classes)
```

## 核心问题2：与Spatial AGI的关系

### Q2：FVG-PT与Spatial AGI的关系是什么？

FVG-PT虽然专注于prompt tuning优化，但其核心思想与Spatial AGI（空间人工智能）研究高度相关，主要体现在以下几个方面：

### 1. 空间注意力机制

#### 核心联系

**Spatial AGI的核心能力**：理解和推理物理世界的空间结构
- 3D场景理解
- 空间关系推理
- 物体定位与识别

**FVG-PT的贡献**：通过前景注意力引导，增强模型的空间感知能力

**具体体现**：
- **前景vs背景**：本质上是空间分割问题，区分图像中的"主要物体"（前景）和"环境"（背景）
- **注意力分布**：模型需要学习"看哪里"，即空间定位能力
- **Grad-CAM可视化**：直接展示了模型的空间注意力模式

#### Spatial AGI视角下的解读

从Spatial AGI的角度，FVG-PT解决的问题可以重新表述为：

**问题**：在prompt tuning过程中，模型的空间注意力定位能力退化
- 从正确的前景位置偏移到无关背景
- 失去了对"主要物体"的空间感知

**解决方案**：通过显式的前景监督，重新引导模型的空间注意力
- 使用SEEM分割模型提供空间先验
- 学习自适应的空间注意力分配
- 保持空间泛化能力（通过PC模块）

### 2. 前景作为空间实体

#### 空间表征

**前景视图 x_fg = x ⊙ m**：
- m是空间mask，定义了图像中的"感兴趣区域"（ROI）
- 前景提取本质上是空间分割任务
- 前景质量直接影响模型的空间理解能力

#### Spatial AGI的启示

**当前FVG-PT**：
- 使用2D语义分割（SEEM）
- 前景定义基于视觉显著性

**Spatial AGI的潜在扩展**：
1. **3D前景提取**：
   - 结合深度信息
   - 使用3D分割或point cloud segmentation
   - 前景定义可以考虑3D空间结构

2. **多视角一致性**：
   - 对于3D场景，多个视角的前景应保持一致
   - 可以引入3D几何约束

3. **空间关系建模**：
   - 不仅识别前景，还建模前景与背景的空间关系
   - 例如：前景物体在背景场景中的位置、朝向等

### 3. 自适应空间注意力

#### FRG的空间意义

**前景可靠性门控**：
- 评估"这个空间区域是否真的包含主要物体？"
- 3个指标都有空间含义：
  1. **ΔH**：前景区域的信息增益
  2. **cos(p_full, p_fg)**：空间一致性的语义约束
  3. **A(m)**：前景区域的几何大小

#### Spatial AGI的应用

**空间不确定性量化**：
- FRG可以看作是空间可靠性的度量
- 在3D场景中，可以扩展为评估不同视角、不同深度的空间可靠性

**自适应空间采样**：
- 根据FRG分数，自适应地关注不同的空间区域
- 类似于attention机制，但基于显式的前景定义

### 4. Base-New Trade-off与空间泛化

#### BNT问题的空间解读

**Base classes**：训练时见过的空间类别（例如：特定场景中的物体）
**New classes**：测试时未见的空间类别

**BNT本质**：
- 过度拟合训练空间分布 → 泛化能力下降
- 过度关注特定前景 → 忽略通用空间特征（如背景中的上下文信息）

#### PC模块的空间意义

**Prior Calibration**：
- 保留CLIP先验 = 保留通用空间理解能力
- 自适应平衡 = 在特定空间任务和通用空间理解之间权衡

**Spatial AGI的视角**：
- 这是在"特定场景理解"和"通用空间推理"之间的平衡
- 对于AGI，需要同时具备：
  1. **场景特定知识**（base classes）
  2. **通用空间推理能力**（new classes）

### 5. 即插即用与模块化Spatial AGI

#### 架构优势

FVG-PT的即插即用特性与Spatial AGI的模块化设计理念一致：

**Spatial AGI的模块化需求**：
- 视觉编码器
- 空间推理模块
- 任务特定模块
- 通用知识模块

**FVG-PT的模块化设计**：
- **Backbone**：可以是任何VLM（CLIP等）
- **FRG**：空间可靠性评估模块
- **FDC**：空间注意力引导模块
- **PC**：通用知识保留模块

#### 兼容性

FVG-PT兼容多种backbone：
- CoOp（文本prompts）
- KgCoOp（知识增强）
- PromptSRC（跨模态prompts）
- MMRL（中间层插件）

**对Spatial AGI的启示**：
- 空间注意力引导可以作为一个独立模块
- 可以与不同的spatial AGI架构结合
- 不破坏原有架构的优化方向

### 6. 潜在的Spatial AGI扩展方向

#### 6.1 3D场景理解

**当前**：2D前景分割
**扩展**：
- 3D点云前景提取
- 多视角前景一致性
- 3D空间注意力引导

#### 6.2 空间关系推理

**当前**：前景分类
**扩展**：
- 前景物体之间的空间关系（左、右、上、下、前、后）
- 前景与背景的空间交互
- 空间场景图（Spatial Scene Graph）

#### 6.3 动态场景

**当前**：静态图像
**扩展**：
- 视频中的时序前景跟踪
- 动态空间注意力
- 运动前景检测

#### 6.4 语义空间

**当前**：视觉前景
**扩展**：
- 语义空间（例如：导航空间、操作空间）
- 任务相关的前景定义
- 交互式前景选择

#### 6.5 具身智能

**当前**：被动观察
**扩展**：
- 主动视角选择（选择能看清前景的视角）
- 交互式前景探索
- 机器人操作中的前景引导

### 7. 与Spatial AGI研究方向的关联

#### 7.1 3D场景重建

**关联**：

## 核心问题3：深度分析与讨论

### Q3：基于Q1和Q2的深度问题

#### 问题3.1：FRG的自适应机制如何工作？为什么需要多个可靠性指标？

**深度分析**：

FRG使用3个指标的组合，而非单一指标，这背后有深刻的考量：

**1. 单一指标的局限性**

假设只使用ΔH（分布指标）：
- **问题**：即使ΔH > 0（前景熵更低），前景mask可能仍然不完整
- **后果**：模型可能学习到不完整的前景模式

假设只使用A(m)（几何指标）：
- **问题**：大面积的前景mask可能包含大量背景噪声
- **后果**：前景质量低，但面积大

**2. 多指标协同**

三个指标从不同角度约束前景质量：

- **ΔH**：从**信息论**角度，评估前景是否更"确定"
- **cos(p_full, p_fg)**：从**分布**角度，防止前景分布偏离完整图像太远
- **A(m)**：从**几何**角度，确保前景不会太小

**3. 自适应权重学习**

MLP学习如何组合这些指标：
- 不同场景下，不同指标的重要性不同
- 例如：
  - **复杂背景**：cos相似度更重要，防止前景包含背景噪声
  - **小物体**：A(m)更重要，防止前景过小
  - **模糊边界**：ΔH更重要，依赖熵来判断

**4. 为什么使用MLP而非固定权重？**

**固定权重的问题**：
- 无法适应不同场景
- 需要人工调参

**MLP的优势**：
- 从数据中学习最优组合
- 可以学习非线性关系
- 类别无关，泛化能力强

**5. 实验验证**

消融实验（Table 6）显示：
- 移除任何单一指标都会导致性能下降
- 下降幅度不大（0.03-0.16），说明模型对单个指标不敏感
- 证明了多指标设计的鲁棒性

**6. Spatial AGI视角**

在Spatial AGI中，这种多指标设计可以扩展为：
- **深度一致性**：多视角下前景深度应一致
- **时序一致性**：视频中前景应平滑变化
- **语义一致性**：前景应符合物体语义边界

#### 问题3.2：FDC为什么要在特征alignment之后插入adapter？而不是在encoder中间？

**深度分析**：

**1. 设计选择对比**

**方案A**：在encoder中间插入adapter（类似VPT, MMRL）
- **优点**：可以影响中间层特征
- **缺点**：
  - 破坏原有encoder的特征提取流程
  - 与backbone的优化方向可能冲突
  - 难以即插即用

**方案B**：在特征alignment之后插入adapter（FDC的选择）
- **优点**：
  - 保持backbone特征提取流程不变
  - 只影响最终的图像-文本交互
  - 即插即用，不破坏原有优化方向
- **缺点**：
  - 对中间层特征无直接影响

**2. 为什么FDC选择方案B？**

**核心理念**：**补偿（Compensation）而非修改（Modification）**

**FDC的目标**：
- 不是改变backbone的特征提取
- 而是在原有特征基础上，学习前景导向的重投影

**类比**：
- Backbone：原相机镜头
- FDC：外加滤镜，改变最终成像，但不改变镜头本身

**3. 理论分析**

设backbone输出特征为f(x)，FDC adapter为h：

**方案A**：f_new(x) = encoder_with_adapter(x)
- 修改了encoder内部结构
- 原有优化方向可能不再适用

**方案B**：f_new(x) = L2Norm(f(x) + h(f(x)))
- f(x)保持不变（backbone冻结）
- h学习残差修正
- 原有优化方向仍然有效（通过残差连接）

**4. 梯度流分析**

**方案B的梯度流**：
```
∂L/∂f = ∂L/∂f_new · (I + ∂h/∂f)
```
- I是单位矩阵（来自残差连接）
- 即使h的梯度不稳定，仍有直接路径到f
- 保证训练稳定性

**5. 即插即用兼容性**

**Backbone兼容性要求**：
- 不修改backbone结构
- 不依赖backbone内部层
- 只使用backbone最终输出

**FDC的设计**：
- Adapter只在backbone输出后工作
- 可以适配任何VLM backbone
- 实验证明：兼容CoOp, KgCoOp, PromptSRC, MMRL

**6. Spatial AGI启示**

在Spatial AGI中，这种设计理念可以应用于：
- **空间模块**：作为后处理模块，而非修改backbone
- **多模态融合**：在特征层面融合，保持各模态encoder独立
- **任务模块**：任务特定模块作为插件，不破坏通用backbone

#### 问题3.3：PC模块如何实现base和new分支的完全解耦？为什么在logit层面解耦？

**深度分析**：

**1. 解耦的必要性**

**BNT问题根源**：
- Base分支：在base classes上优化，可能过度关注特定前景
- New分支：需要泛化到new classes，依赖通用特征
- **冲突**：Base分支的优化方向可能损害New分支的泛化能力

**2. 解耦层级对比**

**方案A**：特征层解耦（例如：DePT, DPC）
- 在特征层面分离base和new分支
- **问题**：仍可能通过共享的encoder产生干扰

**方案B**：Prompt层解耦（例如：DAPT）
- 使用不同的prompts
- **问题**：prompts仍然在同一个encoder中处理

**方案C**：Logit层解耦（FVG-PT的PC模块）
- 在最终预测logits层面完全分离
- **优势**：彻底隔离，无任何共享计算

**3. FVG-PT的Logit层解耦机制**

**推理阶段**：

**Base分支**：
```python
# 使用微调的FDC
z_base = z_FDC(x) = [f_tilde(x)]^T g_tilde(t)
```

**New分支**：
```python
# 完全独立的计算路径
# 1. 使用原始CLIP（未微调）
z_CLIP = [f_CLIP(x)]^T g_CLIP(t_CLIP)

# 2. BRG自适应权重
b = sigmoid(MLP_BRG(s))

# 3. 校准
z_new = (1-b)·z_full + b·z_CLIP
```

**关键点**：
- New分支**不调用FDC**，完全独立
- New分支使用**原始CLIP**，保留通用能力
- 两个分支的logits计算**完全不共享**

**4. 为什么Logit层解耦更彻底？**

**计算图分析**：

**特征层解耦**：
```
x → Encoder → [f_base, f_new] → [Logit_base, Logit_new]
                ↑ 共享部分
```

**Logit层解耦**：
```
Base: x → Encoder_tuned → f_tilde → Logit_base
New:  x → Encoder_CLIP  → f_CLIP  → Logit_new
      ↑ 完全独立的路径
```

**5. 训练时的解耦**

虽然训练时使用相同数据，但优化目标分离：

**Base分支**：
```python
L_base = CE(z_FDC(x), t) + L_FRG + λ_d·L_dist
```
- 优化FDC adapters
- 优化FRG
- 目标：base classes性能

**New分支**：
```python
L_PC = CE(z_PC(x), t) + KL(p_CLIP || softmax(z_PC/τ_d))
```
- 优化BRG
- 目标：在base数据上学习如何平衡backbone和CLIP先验
- **关键**：不优化FDC，只优化如何使用CLIP先验

**6. BRG的作用**

**为什么需要BRG？**

如果直接使用CLIP先验：
```python
z_new = z_CLIP  # 简单方案
```
- **问题**：完全忽略backbone的优化成果
- **结果**：Base和New性能都受影响

**BRG的自适应平衡**：
```python
z_new = (1-b)·z_full + b·z_CLIP
```
- b由BRG学习，根据输入样本自适应
- 有些样本更依赖CLIP先验（b大）
- 有些样本更依赖backbone（b小）

**7. 实验验证**

**Table 3消融实验**：
- (1) 只用FDC，不用PC：New性能下降4.11%
- (2) 只用PC，不用FDC：New性能提升2.02%
- (5) 完整FVG-PT：New性能提升2.67%

**结论**：
- PC模块对New性能至关重要
- Logit层解耦有效保留泛化能力

**8. Spatial AGI启示**

在Spatial AGI中，这种解耦思想可以应用于：
- **已知场景 vs 未知场景**：
  - 已知场景：使用场景特定的优化模块
  - 未知场景：保留通用空间推理能力
- **特定任务 vs 通用能力**：
  - 特定任务：使用任务特定模块
  - 通用能力：保留基础空间感知模块

#### 问题3.4：FVG-PT的自适应性体现在哪些方面？为什么需要自适应？

**深度分析**：

**1. FVG-PT的三重自适应**

**自适应1**：FRG - 前景视图质量的自适应评估
- **输入**：3个可靠性指标
- **输出**：前景信任分数r ∈ (0,1)
- **自适应**：不同样本，不同r

**自适应2**：FDC - 前景蒸馏的自适应目标
- **输入**：前景信任分数r
- **输出**：蒸馏目标（p_fg或p_full的加权）
- **自适应**：r大→更接近p_fg，r小→更接近p_full

**自适应3**：BRG - CLIP先验的自适应平衡
- **输入**：3个统计指标
- **输出**：CLIP信任分数b ∈ (0,1)
- **自适应**：不同样本，不同b

**2. 为什么需要自适应？**

**问题**：固定策略的局限性

**固定策略1**：总是信任前景
- **问题**：前景可能不准确、不完整
- **后果**：学习到错误的alignment

**固定策略2**：总是使用CLIP先验
- **问题**：忽略backbone的优化成果
- **后果**：Base性能下降

**固定策略3**：固定权重（例如：r=0.5, b=0.5）
- **问题**：不同样本需要不同的策略
- **后果**：次优性能

**3. 自适应的必要性来源**

**来源1**：数据异质性
- 不同图像的前景质量不同
- 不同类别的背景复杂度不同

**来源2**：任务多样性
- Base classes：需要特定前景特征
- New classes：需要通用特征

**来源3**：模型状态
- 训练初期：前景可能不可靠
- 训练后期：前景可能更可靠

**4. 自适应的实现机制**

**机制1**：学习型门控
- 使用MLP学习从指标到分数的映射
- 从数据中自动学习最优策略
- 无需人工规则

**机制2**：多指标输入
- 提供丰富的上下文信息
- MLP可以学习复杂的决策边界
- 对噪声鲁棒

**机制3**：连续输出
- r和b都是连续值（0,1）
- 不是硬选择，而是软加权
- 梯度可微，端到端训练

**5. 自适应的行为分析**

**FRG的自适应行为**：
- **高质量前景**（准确、完整）：
  - ΔH大（前景熵低）
  - cos相似度高
  - A(m)合理
  - → r接近1，强依赖前景

- **低质量前景**（不准确、不完整）：
  - ΔH小或负
  - cos相似度低
  - A(m)过小或过大
  - → r接近0，回退到完整图像

**BRG的自适应行为**：
- **CLIP更自信**：
  - H(p_CLIP)低（分布sharp）
  - H(p_full)高（分布flat）
  - → b大，更依赖CLIP

- **Backbone更自信**：
  - H(p_full)低
  - H(p_CLIP)高
  - → b小，更依赖backbone

**6. 实验证据**

**Table 6**：移除任何单一指标，性能下降
- 证明自适应需要多个指标协同

**Figure 4(b)**：不同λ_d的性能
- λ_d=10最优，但其他值也优于baseline
- 说明FVG-PT对超参数鲁棒，自适应能力强

**7. 与非自适应方法对比**

**ProGrad**：基于梯度约束，固定规则
- 无法根据前景质量调整
- 性能不如FVG-PT（Figure 4(a)）

**DAPT-S**：直接使用所有前景线索，无质量评估
- 可能引入噪声
- 性能不如FVG-PT

**8. Spatial AGI启示**

在Spatial AGI中，自适应性更加重要：
- **场景多样性**：不同场景需要不同的空间推理策略
- **任务多样性**：不同任务需要关注不同的空间特征
- **动态环境**：环境变化需要自适应调整

**潜在扩展**：
- **视角自适应**：根据视角质量调整空间推理
- **深度自适应**：根据深度估计可靠性调整
- **时序自适应**：根据运动状态调整

#### 问题3.5：FVG-PT的计算开销如何？为什么能实现参数高效？

**深度分析**：

**1. 参数量分析**

**Backbone (MMRL)**：4.99M可训练参数
**FVG-PT**：0.13M可训练参数

**FVG-PT参数分解**：

**FRG**：
- MLP: 输入3维 → 隐藏32维 → 输出1维
- 参数量: (3×32 + 32) + (32×1 + 1) = 128 + 33 = 161

**FDC Adapters**：
- 视觉adapter: 输入512维 → 隐藏64维 → 输出512维
  - 参数量: (512×64 + 64) + (64×512 + 512) = 32,832 + 33,280 = 66,112
- 文本adapter: 同上，66,112
- 总计: 132,224

**BRG**：
- MLP: 输入3维 → 隐藏32维 → 输出1维
- 参数量: 161

**总计**: 161 + 132,224 + 161 ≈ 132,546 ≈ 0.13M

**2. 为什么参数少？**

**原因1**：瓶颈结构
- Adapter隐藏层64维 << 输入512维
- 参数量从512² ≈ 0.26M降到512×64×2 ≈ 0.066M

**原因2**：门控简单
- FRG和BRG只需学习3→1的映射
- 输入已经是高层统计特征，不需要大容量网络

**原因3**：共享backbone
- 只训练插件模块
- Backbone完全冻结

**3. 计算复杂度分析**

**前向传播**：

**FRG**：
- 计算3个指标: O(1)（已有logits）
- MLP前向: O(3×32 + 32×1) ≈ O(128)

**FDC**：
- 两个adapter: O(512×64×2) ≈ O(65,536)

**BRG**：
- 计算3个指标: O(1)
- MLP前向: O(128)

**总计**: O(65,792) ≈ O(65K)

**对比backbone前向**：
- ViT-B/16: O(n×d²) ≈ O(196×768²) ≈ O(115M)
- FVG-PT额外开销: 65K / 115M ≈ 0.06%

**4. 内存占用分析**

**Table 7**：
- MMRL: 2009.9 MB
- FVG-PT: 808.4 MB

**为什么内存更少？**

**原因1**：参数少
- 可训练参数少 → 梯度占用内存少
- 优化器状态占用少

**原因2**：只激活部分模块
- Base分支只用FDC
- New分支只用BRG
- 不需要同时存储所有中间特征

**5. FPS分析**

**Table 7**：
- MMRL tuning FPS: 87.52
- FVG-PT tuning FPS: 153.36
- MMRL inference FPS: 610.6
- FVG-PT inference FPS: 847.6

**为什么更快？**

**Tuning阶段**：
- FVG-PT参数少 → 反向传播快
- FVG-PT模块简单 → 前向快

**Inference阶段**：
- FVG-PT是轻量级后处理
- 不修改backbone主体计算

**6. 与其他方法对比**

**全量微调**：
- 参数量: ~100M (CLIP ViT-B/16)
- 内存: 极高
- 速度: 慢

**Prompt Tuning (CoOp)**：
- 参数量: ~2K (文本prompts)
- 内存: 低
- 速度: 快

**Adapter方法 (CLIP-Adapter)**：
- 参数量: ~3.5M
- 内存: 中等
- 速度: 中等

**FVG-PT**：
- 参数量: 0.13M
- 内存: 低（808 MB）
- 速度: 快（847 FPS）

**7. 参数高效的实现技巧**

**技巧1**：瓶颈设计
- 所有MLP都使用瓶颈结构
- 隐藏层维度远小于输入输出

**技巧2**：残差连接
- 保留原始特征，只学习修正
- 降低学习难度

**技巧3**：统计特征
- FRG和BRG使用统计特征而非原始特征
- 大幅降低输入维度

**技巧4**：模块化
- 每个模块独立，参数不共享
- 但每个模块都很小

**8. Spatial AGI启示**

在Spatial AGI中，参数高效更加重要：
- **3D处理**：计算量更大
- **多模态**：需要处理多种输入
- **实时性**：机器人应用需要高FPS

**FVG-PT的启示**：
- 使用瓶颈结构减少参数
- 使用统计特征降低维度
- 模块化设计保持灵活性

**潜在扩展**：
- **3D adapter**：瓶颈结构扩展到3D特征

- **实时推理**：优化FVG-PT用于边缘设备

## 方法论详解

### 3.1 预备知识

#### CLIP基础

**架构**：
- 视觉编码器 f(·)：ViT-based
- 文本编码器 g(·)：Transformer-based
- 对比学习：图像-文本alignment

**分类流程**：
```python
# 图像编码
f_image = f(x)  # 图像特征

# 文本编码（每个类别）
for class in classes:
    t_class = "A photo of a " + class
    f_text_class = g(t_class)

# 图像-文本相似度
logits = [sim(f_image, f_text_class) for class in classes]
prediction = argmax(logits)
```

#### Prompt Tuning

**核心思想**：引入可学习prompt向量，而非手工设计prompt

**文本prompts (CoOp)**：
```python
P = [θ₁, θ₂, ..., θ_L]  # 可学习向量
t_class = P + [CLASS]   # 拼接类别token
```

**视觉prompts (VPT)**：
```python
P_v = [θ₁, θ₂, ..., θ_L]
input = P_v + image_patches
```

**损失函数**：
```python
L_CE = -Σ h_i log exp(sim(g(P_t^y), f(P_v, x))/τ) / Z
```

### 3.2 Foreground Reliability Gate (FRG)

#### 动机

**观察**：前景视图质量不可控
- SEEM分割可能不准确
- 前景可能不完整或过大
- 需要自适应评估机制

#### 实现

**Step 1**：前景提取
```python
# 使用SEEM分割模型
m = SEEM_segment(x)  # m ∈ {0,1}^{H×W}
x_fg = x * m  # 逐元素乘法
```

**Step 2**：计算可靠性指标
```python
# 1. 分布指标
p_full = softmax(z(x) / τ_d)
p_fg = softmax(z(x_fg) / τ_d)
ΔH = H(p_full) - H(p_fg)

# 2. 相似性指标
cos_sim = cosine_similarity(p_full, p_fg)

# 3. 几何指标
A_m = sum(m) / (H * W)  # 前景面积比
```

**Step 3**：门控预测
```python
u = [ΔH, cos_sim, A_m]  # 输入向量
q = MLP_FRG(u)          # MLP前向
r = sigmoid(q)          # 信任分数
```

**Step 4**：训练
```python
# 监督信号
l_full = CE(z(x), t)
l_fg = CE(z(x_fg), t)
r* = 1 if l_fg < l_full else 0

# 损失
L_FRG = BCE(r*, r)
```

#### 关键设计

**设计1**：类别无关
- FRG不依赖类别标签
- 可以泛化到任何前景视图

**设计2**：基于backbone logit
- 使用backbone最终输出
- 不依赖内部特征
- 确保即插即用

**设计3**：多指标协同
- 3个指标互补
- 对单一指标噪声鲁棒

### 3.3 Foreground Distillation Compensation (FDC)

#### 动机

**目标**：引导视觉注意力聚焦于前景

**挑战**：
- 不能破坏backbone原有特征
- 需要学习前景导向的重投影
- 需要即插即用

#### 实现

**Step 1**：特征提取
```python
# 使用冻结的backbone
f_image = backbone.f(x)  # 视觉特征
f_text = backbone.g(t)   # 文本特征
```

**Step 2**：Adapter重投影
```python
# 视觉adapter
h_v = MLP_v(f_image)           # 瓶颈MLP
f_tilde = L2Norm(f_image + h_v)  # 残差+归一化

# 文本adapter
h_t = MLP_t(f_text)
g_tilde = L2Norm(f_text + h_t)
```

**Step 3**：FDC logit
```python
z_FDC = f_tilde.T @ g_tilde  # 图像-文本相似度
p_FDC = softmax(z_FDC / τ_d)
```

**Step 4**：蒸馏损失
```python
# 自适应目标
if r > 0.5:  # 前景可靠
    target = p_fg
else:  # 前景不可靠
    target = p_full

# KL散度
L_dist = r * KL(p_fg || p_FDC) + (1-r) * KL(p_full || p_FDC)
```

**Step 5**：总体损失
```python
L_base = CE(z_FDC, t) + L_FRG + λ_d * L_dist
```

#### 关键设计

**设计1**：瓶颈adapter
- 隐藏层64维 << 输入512维
- 参数量：2 × (512×64 + 64×512) ≈ 0.13M

**设计2**：双分支adapter
- 视觉和文本独立
- 避免单边修改破坏alignment

**设计3**：残差连接
- 保留原始特征
- 只学习修正

**设计4**：自适应蒸馏
- 根据r动态调整目标
- 可靠前景→接近p_fg
- 不可靠前景→接近p_full

### 3.4 Prior Calibration (PC)

#### 动机

**BNT问题**：
- 过度关注前景→忽略背景常识
- Base classes性能↑，New classes性能↓

**解决思路**：
- 解耦base和new分支
- 保留CLIP先验用于new分支

#### 实现

**Step 1**：New分支计算
```python
# 使用原始CLIP（未微调）
f_CLIP = CLIP.f(x)  # CLIP视觉特征
g_CLIP = CLIP.g(t_CLIP)  # CLIP文本特征（"A photo of a [CLASS]"）
z_CLIP = f_CLIP.T @ g_CLIP
```

**Step 2**：BRG计算
```python
# 统计指标
p_full = softmax(z_full / τ_d)
p_CLIP = softmax(z_CLIP / τ_d)

s = [H(p_full), H(p_CLIP), cosine_similarity(p_full, p_CLIP)]

# MLP预测
b = sigmoid(MLP_BRG(s))
```

**Step 3**：校准logits
```python
z_PC = (1-b) * z_full + b * z_CLIP
```

**Step 4**：训练损失
```python
L_PC = CE(z_PC, t) + KL(p_CLIP || softmax(z_PC/τ_d))
```

#### 关键设计

**设计1**：Logit层解耦
- New分支不调用FDC
- 使用独立的CLIP路径
- 完全隔离优化

**设计2**：自适应平衡
- BRG学习样本特定的b
- 不同样本依赖不同比例的CLIP/backbone

**设计3**：KL正则
- 保持校准分布接近CLIP
- 防止过度偏离先验

### 3.5 训练流程

**总体损失**：
```python
L_total = L_base + L_PC
```

**训练步骤**：
1. 前向传播：
   - 计算前景视图x_fg
   - FRG预测r
   - FDC计算z_FDC
   - BRG预测b
   - PC计算z_PC

2. 损失计算：
   - L_FRG = BCE(r*, r)
   - L_dist = r·KL(p_fg||p_FDC) + (1-r)·KL(p_full||p_FDC)
   - L_base = CE(z_FDC, t) + L_FRG + λ_d·L_dist
   - L_PC = CE(z_PC, t) + KL(p_CLIP||softmax(z_PC/τ_d))

3. 反向传播：
   - 更新FRG参数
   - 更新FDC adapter参数
   - 更新BRG参数

**优化器**：SGD (CoOp, KgCoOp, PromptSRC) 或 AdamW (MMRL)
**学习率**：0.0035
**Epochs**：10

### 3.6 推理流程

**输入**：图像x

**Base分支**：
```python
f_tilde = L2Norm(f(x) + MLP_v(f(x)))
g_tilde = L2Norm(g(t) + MLP_t(g(t)))
z_base = f_tilde.T @ g_tilde
y_base = argmax(z_base)
```

**New分支**：
```python
z_CLIP = CLIP.f(x).T @ CLIP.g(t_CLIP)
b = sigmoid(MLP_BRG(s))
z_new = (1-b)*z_full + b*z_CLIP
y_new = argmax(z_new)
```

**输出**：
- Base classes: y_base
- New classes: y_new

### A.1 前景注意力偏移的定量分析

#### 前景偏移指数

**定义**：
```python
D_fg = KL(p_full || p_fg)
```

- p_full: 完整图像的预测分布
- p_fg: 前景视图的预测分布
- D_fg: 前景一致性（KL散度）

**解释**：
- D_fg小：模型主要依赖前景进行预测（前景一致性高）
- D_fg大：模型关注背景或前景质量差（前景一致性低）

#### 实验结果 (Figure 5)

**数据集**：Caltech101, Flowers102

**模型对比**：
1. CLIP (基础模型)
2. CoOp (fine-tuned)
3. FVG-PT (ours)

**趋势**：
- Base性能↑ → D_fg↓
- CLIP → CoOp → FVG-PT
- 准确率提升，前景偏移减小

**结论**：
- 前景注意力偏移与性能负相关
- FVG-PT有效纠正偏移

## 局限性与未来工作

### 5.1 当前局限

#### 局限1：视觉专有方法不兼容

**问题**：
- FVG-PT不适用于纯视觉prompt tuning（如VPT）
- 这些方法缺乏文本分支

**原因**：
- FDC需要在视觉和文本分支都插入adapter
- 缺少文本分支无法实现跨模态补偿

**影响范围**：
- VPT (Visual Prompt Tuning)
- 其他视觉only方法

#### 局限2：知识蒸馏方法的成本

**问题**：
- 对于使用全部无标注图像的方法（如PromptKD）
- 获取所有图像的前景视图成本高

**原因**：
- SEEM分割需要逐图像处理
- 大规模数据集（如ImageNet-1K全部图像）成本显著

**影响范围**：
- PromptKD
- 其他使用大规模无标注数据的方法

#### 局限3：2D限制

**问题**：
- 当前只处理2D图像
- 未考虑3D场景、深度信息

**原因**：
- SEEM是2D分割模型
- CLIP backbone基于2D图像

**影响范围**：
- 3D视觉任务
- 机器人导航、操作等需要3D理解的任务

#### 局限4：静态场景限制

### A. FRG实现（伪代码）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class ForegroundReliabilityGate(nn.Module):
    def __init__(self, hidden_dim=32):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(3, hidden_dim),  # 3个指标
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )
    
    def compute_indicators(self, z_full, z_fg, mask):
        """
        计算可靠性指标
        Args:
            z_full: 完整图像logits [B, C]
            z_fg: 前景视图logits [B, C]
            mask: 前景mask [B, H, W]
        Returns:
            u: 指标向量 [B, 3]
        """
        tau_d = 2.0
        
        # 分布
        p_full = F.softmax(z_full / tau_d, dim=-1)
        p_fg = F.softmax(z_fg / tau_d, dim=-1)
        
        # 指标1：熵差
        H_full = -torch.sum(p_full * torch.log(p_full + 1e-8), dim=-1)
        H_fg = -torch.sum(p_fg * torch.log(p_fg + 1e-8), dim=-1)
        delta_H = H_full - H_fg
        
        # 指标2：余弦相似度
        cos_sim = F.cosine_similarity(p_full, p_fg, dim=-1)
        
        # 指标3：面积比
        A_m = mask.sum(dim=(1,2)) / (mask.shape[1] * mask.shape[2])
        
        # 组合
        u = torch.stack([delta_H, cos_sim, A_m], dim=-1)
        return u
    
    def forward(self, z_full, z_fg, mask):
        """
        前向传播
        Returns:
            r: 前景信任分数 [B]
        """
        u = self.compute_indicators(z_full, z_fg, mask)
        q = self.mlp(u).squeeze(-1)
        r = torch.sigmoid(q)
        return r

# 训练
def train_frg(frg, backbone, images, labels, masks):
    """
    训练FRG
    """
    optimizer = torch.optim.SGD(frg.parameters(), lr=0.0035)
    
    for epoch in range(10):
        for img, label, mask in zip(images, labels, masks):
            # 前景视图
            img_fg = img * mask.unsqueeze(1)
            
            # Backbone预测
            with torch.no_grad():
                z_full = backbone(img)
                z_fg = backbone(img_fg)
            
            # 计算监督信号
            l_full = F.cross_entropy(z_full, label)
            l_fg = F.cross_entropy(z_fg, label)
            r_star = (l_fg < l_full).float()
            
            # FRG预测
            r = frg(z_full, z_fg, mask)
            
            # 损失
            loss = F.binary_cross_entropy(r, r_star)
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
```

### B. FDC实现（伪代码）

```python
class FDCAdapter(nn.Module):
    def __init__(self, input_dim=512, hidden_dim=64):
        super().__init__()
        self.adapter = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, input_dim)
        )
    
    def forward(self, x):
        """
        Args:
            x: 输入特征 [B, D]
        Returns:
            x_tilde: 重投影特征 [B, D]
        """
        residual = self.adapter(x)
        x_tilde = F.normalize(x + residual, p=2, dim=-1)
        return x_tilde

class ForegroundDistillationCompensation(nn.Module):
    def __init__(self, visual_dim=512, text_dim=512, hidden_dim=64):
        super().__init__()
        self.visual_adapter = FDCAdapter(visual_dim, hidden_dim)
        self.text_adapter = FDCAdapter(text_dim, hidden_dim)
    
    def forward(self, f_visual, f_text):
        """
        Args:
            f_visual: 视觉特征 [B, D_v]
            f_text: 文本特征 [B, D_t]
        Returns:
            z_fdc: FDC logits [B, C]
        """
        f_tilde = self.visual_adapter(f_visual)
        g_tilde = self.text_adapter(f_text)
        
        z_fdc = torch.matmul(f_tilde, g_tilde.T)
        return z_fdc

# 训练
def train_fdc(fdc, frg, backbone, images, labels, masks):
    """
    训练FDC
    """
    optimizer = torch.optim.SGD(fdc.parameters(), lr=0.0035)
    lambda_d = 10.0
    tau_d = 2.0
    
    for epoch in range(10):
        for img, label, mask in zip(images, labels, masks):
            # 前景视图
            img_fg = img * mask.unsqueeze(1)
            
            # Backbone特征
            with torch.no_grad():
                f_visual = backbone.encode_image(img)
                f_text = backbone.encode_text(label)
                z_full = backbone(img)
                z_fg = backbone(img_fg)
            
            # FRG分数
            r = frg(z_full, z_fg, mask)
            
            # FDC logits
            z_fdc = fdc(f_visual, f_text)
            
            # 蒸馏目标
            p_full = F.softmax(z_full / tau_d, dim=-1)
            p_fg = F.softmax(z_fg / tau_d, dim=-1)
            p_fdc = F.log_softmax(z_fdc / tau_d, dim=-1)
            
            # KL散度
            kl_fg = F.kl_div(p_fdc, p_fg, reduction='batchmean')
            kl_full = F.kl_div(p_fdc, p_full, reduction='batchmean')
            L_dist = r * kl_fg + (1 - r) * kl_full
            
            # 总损失
            L_ce = F.cross_entropy(z_fdc, label)
            loss = L_ce + lambda_d * L_dist
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
```

### C. PC实现（伪代码）

```python
class BackboneReliabilityGate(nn.Module):
    def __init__(self, hidden_dim=32):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(3, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )
    
    def compute_statistics(self, z_full, z_clip):
        """
        计算统计指标
        """
        tau_d = 2.0
        
        p_full = F.softmax(z_full / tau_d, dim=-1)
        p_clip = F.softmax(z_clip / tau_d, dim=-1)
        
        # 熵
        H_full = -torch.sum(p_full * torch.log(p_full + 1e-8), dim=-1)
        H_clip = -torch.sum(p_clip * torch.log(p_clip + 1e-8), dim=-1)
        
        # 余弦相似度
        cos_sim = F.cosine_similarity(p_full, p_clip, dim=-1)
        
        s = torch.stack([H_full, H_clip, cos_sim], dim=-1)
        return s
    
    def forward(self, z_full, z_clip):
        """
        Returns:
            b: CLIP信任分数 [B]
        """
        s = self.compute_statistics(z_full, z_clip)
        q = self.mlp(s).squeeze(-1)
        b = torch.sigmoid(q)
        return b

class PriorCalibration(nn.Module):
    def __init__(self):
        super().__init__()
        self.brg = BackboneReliabilityGate()
    
    def forward(self, z_full, z_clip):
        """
        Returns:
            z_pc: 校准logits [B, C]
        """
        b = self.brg(z_full, z_clip)
        z_pc = (1 - b.unsqueeze(-1)) * z_full + b.unsqueeze(-1) * z_clip
        return z_pc

# 训练
def train_pc(pc, backbone, clip_model, images, labels):
    """
    训练PC
    """
    optimizer = torch.optim.SGD(pc.parameters(), lr=0.0035)
    tau_d = 2.0
    
    for epoch in range(10):
        for img, label in zip(images, labels):
            # Backbone logits
            with torch.no_grad():
                z_full = backbone(img)
            
            # CLIP先验
            with torch.no_grad():
                z_clip = clip_model(img)
            
            # PC校准
            z_pc = pc(z_full, z_clip)
            
            # 损失
            L_ce = F.cross_entropy(z_pc, label)
            
            p_clip = F.softmax(z_clip / tau_d, dim=-1)
            p_pc = F.log_softmax(z_pc / tau_d, dim=-1)
            L_kl = F.kl_div(p_pc, p_clip, reduction='batchmean')
            
            loss = L_ce + L_kl
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
```

### D. 完整FVG-PT训练流程

```python
class FVGPT(nn.Module):
    def __init__(self, backbone, clip_model):
        super().__init__()
        self.backbone = backbone
        self.clip_model = clip_model
        
        self.frg = ForegroundReliabilityGate()
        self.fdc = ForegroundDistillationCompensation()
        self.pc = PriorCalibration()
    
    def forward(self, img, mask, label):
        """
        完整前向传播
        """
        # 前景视图
        img_fg = img * mask.unsqueeze(1)
        
        # Backbone特征
        f_visual = self.backbone.encode_image(img)
        f_text = self.backbone.encode_text(label)
        z_full = self.backbone(img)
        z_fg = self.backbone(img_fg)
        
        # Base分支
        r = self.frg(z_full, z_fg, mask)
        z_fdc = self.fdc(f_visual, f_text)
        
        # New分支
        z_clip = self.clip_model(img)
        z_pc = self.pc(z_full, z_clip)
        
        return z_fdc, z_pc, r
    
    def compute_loss(self, z_fdc, z_pc, r, z_full, z_fg, label):
        """
        计算总损失
        """
        tau_d = 2.0
        lambda_d = 10.0
        
        # FRG损失
        l_full = F.cross_entropy(z_full, label)
        l_fg = F.cross_entropy(z_fg, label)
        r_star = (l_fg < l_full).float()
        L_frg = F.binary_cross_entropy(r, r_star)
        
        # FDC损失
        p_full = F.softmax(z_full / tau_d, dim=-1)
        p_fg = F.softmax(z_fg / tau_d, dim=-1)
        p_fdc = F.log_softmax(z_fdc / tau_d, dim=-1)
        
        kl_fg = F.kl_div(p_fdc, p_fg, reduction='batchmean')
        kl_full = F.kl_div(p_fdc, p_full, reduction='batchmean')
        L_dist = r * kl_fg + (1 - r) * kl_full
        
        L_ce_fdc = F.cross_entropy(z_fdc, label)
        L_base = L_ce_fdc + L_frg + lambda_d * L_dist
        
        # PC损失
        z_clip = self.clip_model(label)  # 简化
        L_ce_pc = F.cross_entropy(z_pc, label)
        
        p_clip = F.softmax(z_clip / tau_d, dim=-1)
        p_pc = F.log_softmax(z_pc / tau_d, dim=-1)
        L_kl = F.kl_div(p_pc, p_clip, reduction='batchmean')
        
        L_pc = L_ce_pc + L_kl
        
        # 总损失
        L_total = L_base + L_pc
        return L_total

# 训练
def train_fvgpt(fvgpt, images, labels, masks, epochs=10):
    """
    训练FVG-PT
    """
    optimizer = torch.optim.SGD(fvgpt.parameters(), lr=0.0035)
    
    for epoch in range(epochs):
        for img, label, mask in zip(images, labels, masks):
            # 前向传播
            z_fdc, z_pc, r = fvgpt(img, mask, label)
            
            # Backbone logits（用于FRG）
            with torch.no_grad():
                z_full = fvgpt.backbone(img)
                img_fg = img * mask.unsqueeze(1)
                z_fg = fvgpt.backbone(img_fg)
            
            # 损失计算
            loss = fvgpt.compute_loss(z_fdc, z_pc, r, z_full, z_fg, label)
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            print(f"Epoch {epoch}, Loss: {loss.item():.4f}")
```
